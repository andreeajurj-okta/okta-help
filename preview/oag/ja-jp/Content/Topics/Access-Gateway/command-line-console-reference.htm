Command Line Management Consoleのリファレンス

Access Gateway ManagementコンソールはAccess Gatewayインスタンスへのコマンドライン インターフェイスであり、Admin UIコンソールからはアクセスできない構成を変更できます。

このリファレンスの目的は、oag-mgmtユーザーアカウント内で現在利用可能なAccess Gateway Managementコンソールコマンドの一覧を提供することです。これらのコマンドを使用してAccess Gateway Managementを構成および監視できます。

Command Line Management Consoleでは以下のメニューが利用できます：

  • Kerberos

Note

To avoid publication of public IP addresses that belong to Okta or any other organization, the IPv4 addresses used in Okta Access Gateway technical documentation are fictional and are from the private IP address range based on CIDR 192.168/16.

初回ログイン

On successful password change, you will be redirected to the Access Gateway Management console main page.

Network（ネットワーク）

Network（ネットワーク）メニューにはネットワークのステータスを確認したりネットワーク設定を変更するオプションが含まれています。

ネットワークコマンドの概要

  1. Manage network interfaces（ネットワークインターフェイスの管理）：このサブメニューでAccess Gatewayアプライアンスのルーティングなど、ネットワークインターフェイスを管理します。
  2. Test network configuration（ネットワーク構成のテスト）：このサブメニューで複数のWebサイトへの接続を試み、NGINX構成およびステータスを確認します。
  3. Edit /etc/hosts（/etc/hostsの編集）：Access Gatewayアプライアンスの/etc/hostsファイルを編集できます。
  4. Setup NIC bonding（NICボンディングのセットアップ）：スタティックネットワーク構成と同様のNICボンディング構成を行うサブメニューに進みます。
  5. Proxy settings（プロキシ設定）：このサブメニューでAccess Gatewayアプライアンスのプロキシ接続をセットアップしたり、アプライアンスからプロキシ設定を削除したりします。
  6. Ping: 接続先ホストがAccess Gatewayアプライアンスから接続可能か判断します。
  7. Connectivity test（接続テスト）: Access Gatewayアプライアンスや他のシステムとの間の接続を検証します。このツールを使用してバックエンドアプリケーションまたはサーバーがAccess Gatewayに到達できるか検証します。
  8. Manage DNS Settings（DNS設定の管理） - プライマリおよびセカンダリDNSサーバーなどのDNSを設定します。
  9. Manage Trusted Domains（信頼できるドメインの管理） - 信頼できるドメインの有効化、無効化、および表示を行います。
  10. s — 実行中の構成を表示：現在のAccess Gatewayアプライアンスの構成を表示します。
  11. v — 構成変更プランを表示：構成変更プランを表示します。
  12. c — 変更をシステムで確定：保存されていない変更をAccess Gatewayアプライアンスで確定します。
  13. r — ネットワーキングの再起動：Access Gatewayアプライアンスでネットワークサービスを再起動します。
  14. x — 終了：マネジメント コンソールを終了します。

Test network configuration（ネットワーク構成のテスト）

  1. 3を押してネットワーク構成のテストを開始します。

  2. 次に、システムがwww.okta.comへの接続を試み、ステータスを表示します。Enterキーを押して続行します。

  3. 最後に、システムがNGINX構成を確認してステータスを表示します。Enterキーを押してNetworking（ネットワーキング）のメニューに戻ります。

Edit the /etc/hosts file（/etc/hostsファイルの編集）

/etc/hostsは、静的に割り当てられたホスト名を構成するために使用されます。管理者が特定のホスト名のDNSを優先する必要がある場合、またはDNSに存在しないホストのアドレスを提供する必要がある場合、そのホストを/etc/hostsファイルに追加できます。通常、これはOkta Access Gatewayが正しいホストに接続できるようにするために実施されます。

  1. 4を押して/etc/hostsファイルを編集します。これはホストの現在のエントリを行番号ごとに表示します。

  2. aを押して新しいエントリを追加します。

  3. 以下の標準フォーマットに従ってエントリを追加します。

  4. エントリを削除するには、dを押して削除したいエントリに対応する行番号を入力します。

  5. cを押してホストファイルに変更をコミットします。

  6. xを押してNetworking（ネットワーキング）のメインメニューに戻ります。

Setup NIC bonding（NICボンディングのセットアップ）

NIC Bonding is the process of combining two ethernet ports together into a bonded virtual port. Typically, NIC Bonding is used if there is sufficient traffic on a single port to saturate a single network connection.

  1. Enter 5 to set up NIC bonding.

  2. Enter the IP address, and press the Enter key.

  3. Enter the netmask value and press the Enter key.

  4. Enter the default gateway value and press the Enter key. .

  5. Enter the primary DNS server value and press the Enter key.

  6. Enter the secondary DNS server value and press the Enter key.

  7. Enter the search domain address and press the Enter key.

  8. Press c to commit the changes.

  9. Review the network configuration and enter y and press the Enter key to confirm the changes. Alternatively, enter n and press the Enter key to discard your changes and return to the Networking menu.

  10. After the change is complete, enter any key to return to the Networking menu.

プロキシ設定

プロキシの設定

  1. 7を押してインターネットへのプロキシ接続を設定または無効にします。
  2. 1を押してプロキシを構成します。
  3. プロキシのホスト名を入力し、Enterキーを押します。
  4. プロキシポートを入力し、Enterキーを押します。
  5. 必要に応じてプロキシのユーザー名を入力し、Enterキーを押します。
  6. 必要に応じてプロキシのパスワードを入力し、Enterキーを押します。
  7. プロキシをバイパスする必要があるホスト名を入力します（カンマ区切り）。
    入力したプロキシ設定が表示されます。
  8. yを押してプロキシ設定を確定するか、Nを押して処理を中断します。
  9. Enterキーを押してプロキシメニューに戻り、Enterキーを再度押してNetworking（ネットワーキング）のメインメニューに戻ります。

プロキシの設定解除

  1. 7を押してProxi（プロキシ）メニューに進みます。

  2. 2を押してプロキシ構成を削除します。

  3. yを押して確定します。

  4. Enterキーを押してProxi（プロキシ）メニューに戻り、もう一度Enterキーを押してNetworking（ネットワーキング）のメインメニューに戻ります。

Ping

Pingオプションを使用して、接続先ホストがAccess Gatewayアプライアンスから到達可能か確認できます。

  1. 8を押してpingメニューに進みます。

  2. pingしたい接続先ホストのホスト名かIPアドレスを入力し、Enterキーを押します。

  3. システムにpingの結果が表示されます。

  4. Enterキーを押してNetworking（ネットワーキング）のメインメニューに戻ります。

Connectivity test（接続テスト）

接続テストを使用して、Access Gatewayアプライアンスと他のシステムとの間の接続を検証できます。また、このツールを使用してバックエンドアプリケーションまたはサーバーがAccess Gatewayから到達できるかも検証できます。

  1. Network（ネットワーク）メニューから9を選択します。

  2. テストしたいマシンのホスト名かIPアドレスを入力し、Enterキーを押します。

  3. テストを必要とするポート番号を入力し、Enterキーを押します。

  4. システムに接続結果が表示されます。

  5. Enterキーを押してNetworking（ネットワーキング）のメニューに戻ります。

Manage DNS Settings（DNS設定の管理）のサブメニューを使用して、ドメイン名サーバーの追加、削除および変更、ならびにドメインの検索が実施できます。

Manage DNS servers（DNSサーバーの管理）

  1. 1を入力してManage DNS servers（DNSサーバーの管理）のサブメニューに進みます。
    現在のDNS サーバーが以下のように表示されます：
     Managing DNS Servers Primary: 192.168.8.8 Secondary: Tertiary: . . .
  2. 1を入力して第1DNSサーバーを設定します。
  3. 2を入力して第2DNSサーバーを設定します。
  4. 3を入力して第3DNSサーバーを設定します。

    メモ

    DNS サーバーが何も設定されていない場合、第2または第3DNSサーバーの設定をするとそれが第1DNS サーバーに設定されます。
    第1DNS サーバーのIPアドレスは空白にできませんが、第2、第3DNS サーバーは省略できます。

  5. 選択したDNSサーバーのIPアドレスを入力してEnterキーをクリックします。
    アップデートされたDNSサーバーの一覧が表示されます。
  6. xを入力すると終了して親メニューに戻ります。

Manage search domains（検索ドメインの管理）

    1. 新しい検索ドメインを入力してEnterキーを押します。

       Current search domains: okta.com someotherdomain.com . . .

      の削除後：Current search domains: okta.com . . .

Manage Trusted Domains（信頼できるドメインの管理）

Manage Trusted Domains（信頼できるドメインの管理）のメニューは、信頼できるドメインの有効化、無効化、および表示を行うために使用されます。

  1. Network（ネットワーク）メニューから9を選択します。Manage Trusted Domains（信頼できるドメインの管理）のサブメニューが以下のように表示されます：

     Manage Trusted Domains (status: Enabled/Disabled) 1 - Enable/Disable trusted domain 2 - View trust domains x - Exit

  2. xを入力すると終了して親メニューに戻ります。

信頼できるドメインを有効にする、または表示する

  1. 1を入力して信頼できるドメイン有効化／無効化設定を切り替えます。
  2. Manage Trusted Domains（信頼できるドメインの管理）サブメニューが再表示され、信頼できるドメインの現在のステータスが示されます。

信頼できるドメインを表示

  1. 2を押して信頼できるドメインを表示します。 以下のように、現在信頼できるドメインの一式が表示されます：
     UP/DOWN/HOME/END - scroll list x - exit trusted-one.domain.com trusted-two.domain.com . . . trusted-n.domain.com . . .

    メモ

    表示オプションは存在しますが、無効化されている時に信頼できるドメインの表示を試みると警告が表示されます。

  2. [x]を入力して表示を終了します。
    Manage Trusted Domains（信頼できるドメインの管理）サブメニューが再表示され、信頼できるドメインの現在のステータスが示されます。

Show a running configuration（実行中の構成を表示）

実行中の構成を表示するオプションを使用して、インターフェイス、DNS、ルーティング、ホストファイルの内容など、現在の構成の詳細を表示することができます。

  4. Session Cache（セッションキャッシュ）：セッションキャッシュサブメニューを開きます。

サービスコマンド

NGIX

NGINXメニューを使用して、NGINXサービスの開始、停止、再起動、サービスのステータス確認、およびSSL証明書の再生成を行うことができます。

  1. Start（開始）：サービスを開始します。

  2. Stop（停止）：サービスを停止します。

  3. Restart（再起動）：サービスを再起動します。

  4. Check status（ステータスをチェック）：このオプションを使用して、サービスが実行中か停止中かなど、サービスの現在のステータスを表示できます。

  5. Re-generate SSL（SSL証明書を再生成）：SSL 証明書を再生成します。

  6. Update SSLcertificate（SSL証明書をアップデート）：SSL 証明書をアップデートします。

  7. Disable TLS 1.1（TLS 1.1を無効化）：TLS 1.1のサポートを無効にします。

  8. Enable TLS 1.1（TLS 1.1を有効化）：TLS 1.1のサポートを有効にします。

  9. Manage Ciphers（暗号を管理）：暗号を管理します。

Start（開始）、Stop（停止）、Restart（再起動）

  1. NGINXメニューで1、2、または3を押します。

    2. NGINXサービスが要求された操作を実行します。

  2. Enterキーを押してNGINXメニューに戻ります。

Check Status（ステータスをチェック）

  1. NGINXメニューで4を押します。

    2. NGINXサービスステータスが表示されます。

  2. Enterキーを押してNGINXメニューに戻ります。

Regenerate SSL Certificate（SSL証明書を再生成）

  1. NGINXメニューで5を押します。

  2. Access Gatewayアプライアンスのセットアップに使用するドメイン名を入力し、Enterキーを押します。

  3. Enterキーを押してNGINXメニューに戻ります。

Update SSL certificate（SSL証明書を更新）

  1. NGINXメニューで6を押します。

  2. 既知の証明書の一覧が表示されます。

    Enterキーを押して続行します。

  3. 更新する証明書のドメイン名を入力します。
    メモ：置換する証明書ファイルの内容が手元にあり、エディターにペーストする準備ができていなければなりません。

  4. 画面の指示に従い、証明書の内容を挿入して変更し、更新された証明書を保存します。

  5. Enterキーを押してNGINXメニューに戻ります。

Disable or Enable TLS 1.1（TLS 1.1を無効化／有効化）

  1. 7または8を押してNGINXメニューでTLS 1.1を無効／有効にします。

  2. Enterキーを押してNGINXメニューに戻ります。

Manage Ciphers（暗号を管理）

  1. NGINXメニューで9を押します。

    2. メモ：現在指定されている暗号セットが表示されます

  2. 新しい暗号セットを入力します。
    メモ：暗号のセットは「:」で区切られます。「!」のプレフィックスが付いた暗号セットは除外されます。

  3. Enterキーを押してNGINXメニューに戻ります。

Access Gateway Admin

Access Gateway Adminメニューでは、Access Gateway Adminサービスの開始、停止、および再起動や、サービスのステータスを確認できます。

サブメニューには以下のコマンドが含まれています：

  1. Start（開始）：Access Gateway Adminを開始します。

  2. Stop（停止）：Access Gateway Adminを停止します。

  3. Restart（再起動）：Access Gateway Adminを再起動します。

注意

実行中のAccess Gatewayシステムのデバッグを有効にすると、ログレベルがINFOからDEBUGに引き上げられ、生成されるログメッセージの数が大幅に増加します。
デバッグモニタリングの有効化は、アプリケーションおよびサービスをデバッグする場合のみに限定するべきです。デバッグモニタリングの無効化に失敗すると、ディスクの容量不足とシステム性能低下を引き起こすおそれがあります。システムまたはアプリケーションエラーを診断した後は、必ずデバックステータスを無効（INFO）に戻してください。

Support Connection（接続サポート）

The Support Connection menu option allows you to enable and disable support connections, as well as VPN status (enabled or disabled), which will resemble:

Support Connection...
Status: Disabled/Enabled

Support Connection command overview

Enabling or disabling the support connection is a 'toggle' operation. When the support connection is enabled, only disable is displayed. Conversely, when the support connection is disabled, only enable is displayed.

Enter:

  1. Enable Support Connection: Enable support connection functionality
  2. Disable Support Connection:Disable support connection functionality.

Enable support connection

Enabling the support connection requires an API token.
To obtain an API token:

  1. Sign in to your okta org as an administrator.
  2. Select Security > API.
  3. Click Create Token.
  4. Name the token, for example Access Gateway 2021.02 token and click Create Token.
  5. Copy the token value, and then click OK, got it.

  1. Press 1 to enable, or x to exit and return to the prior menu. .

  2. Press y to confirm, or n to cancel.

  3. Enter your Okta org as either {yourorg}.oktapreview.com or {yourorg}.okta.com as appropriate.
  4. Paste the API token obtained as described in the associated note.
    Access Gateway will then validate the token and enabled the support connection.
    Validation and enabling the support connection can anywhere from 1-2 to 20 seconds and is tracked by a progress indicator.
  5. The support connection is now enabled. Press x to return to the prior menu or 2 to disable.

Disable support connection

  1. Press 2 to disable, or x to exit and return to the prior menu. .

  2. Press y to confirm, or n to cancel.

  3. The support connection is now disabled. Press x to return to the prior menu or 1 to enable.

Client certificate chains（クライアント証明書チェーン）

Client certificate chains（クライアント証明書チェーン）メニューのオプションでは、クライアントのCertificate Revocation List (CRL)（証明書失効リストの設定）を表示、追加、削除、その他管理できます。 詳しくは「証明書チェーンの操作」をご覧ください。

証明書チェーンの表示

  1. cを押して証明書チェーン管理メニューに進みます。
  2. 上／下矢印を使用して証明書チェーンを選択します。
    i／キーを使用してリストを上下にスクロールします。
  3. Enterキーを押して選択した証明書を表示します。
  4. xを押してメインメニューに戻ります。

証明書チェーンを追加

  1. cを押して証明書チェーン管理メニューに進みます。
  2. aを押して証明書チェーンを追加します。
  3. 証明書をコマンドウィンドウに貼り付け、完了したら[Ctrl-d]を押します。

    無効または有効期限の切れたチェーンはエラーとなり、読み込まれません。

証明書チェーンを削除

Access Gateway can:

  • Delete entire certificate chains - remove a certificate chain completely, including all intermediary and child certificates.
