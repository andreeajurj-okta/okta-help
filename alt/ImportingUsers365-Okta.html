Importing Users into Okta from an existing Office 365 Tenant

In cases where you have an existing Office 365 tenant containing user and group information, Okta facilitates the import of information to enable these objects to be managed from within Okta.

Benefits

Benefits of importing this information into Okta include:

  • User lifecycle management from within Okta (e.g. updates to personal information and user deactivation)

  • License Management (e.g. assigning additional licenses and group license assignment)

  • Synchronization of group information into other connected systems and services

Once you have set up the Office 365 application and have enabled Provisioning, you can use the Import function from the Application menu to enable this functionality, as described below:

Importing Users

  1. To get started, open the Microsoft Office 365 application you have configured from the Applications menu, then select Import:

  2. Select the Import Now and wait for the Import to complete:

  3. If Okta has been populated with users previously (either from AD, LDAP or directly within Okta), you may choose to join these Office 365 users with existing identities:

    1. Select the Exact option in the Okta User Assignment

    2. Select the Checkbox

    3. Select Confirm Assignments to join these identities (Office 365 to Okta)

  4. If Okta has not been populated previously, these users should be imported and confirmed as new users:

    1. Select the New option in the Okta User Assignment

    2. Select the Checkbox

    3. Select Confirm Assignments to create a new identity in Okta
