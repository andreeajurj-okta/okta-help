iOS デバイスのOkta Verifyで認証する

デバイスのOkta Verifyアプリを使用して本人確認を行うと、Oktaで保護されたアプリのアカウントに安全にサインインすることができます。

開始する前に

デバイスにOkta Verifyの最新のバージョンがセットアップされていることを確認します。詳細については、「iOSデバイスでOkta Verifyをセットアップする」をご覧ください。

デバイスがサインイン方法として構成されている場合、Okta Mobileからアプリにアクセスすることはできません。

Okta Verifyにすでに登録済みのiOSデバイスでChromeのシークレットタブからOkta Verifyを開こうとすると、予期したとおりにOkta Verifyが開く代わりに、[Download Okta Verify (Okta Verifyをダウンロードする)]ページに移動します。

手順

  • Okta Verifyで認証する
  • Okta Verifyで生成されたコードを使用してサインインする
  • プッシュ通知を使用してサインインする
  • デバイスを使用してサインインする
  • Apple Watchで認証する
  • 通知アクションで認証する

Okta Verifyで認証する

IT 部門の構成およびデバイスのセットアップによると、次の異なる方法でOkta Verifyを認証することができます。 

  • Authentication Code (認証コード): Okta Verifyが生成したコードを使用します。
  • Push Notification (プッシュ通知): プッシュ通知で[Yes (はい)]をタップします。
  • This Device (このデバイス): デバイスにより本人確認をします（組織でサポートされている場合）。プロンプトが表示されたら、[Sign in with Okta Verify on this device (このデバイスでOkta Verifyにサインインする)]をタップします。
    利用可能なサインインオプションを確認するには、デバイスでOkta Verifyを開き、アカウントをクリックします。[Account Details (アカウントの詳細)]ページで、サポートするサインイン方法のチェックマークがオンになっています。

デバイスがサインインの方法として構成されており、デバイスで生体認証を変更またはリセットする場合、認証要素が動作していなくても、認証要素のリストで[Use Okta Verify on this device (このデバイスでOkta Veifyを使用する)]オプションが表示されたままとなります。この問題を解決するには、生体認証を無効にしてからもう1度有効にします。ヘルプが必要な場合は、IT部門にお問い合わせください。

Okta Verifyで生成されたコードを使用してサインインする

  1. コンピューターのウェブブラウザーで、組織のOktaアカウントにサインインするか、Oktaで保護されたアプリにアクセスします。
    [Okta Verify Authentication (Okta Verify認証)] チャレンジ画面が表示されます。
  2. [Or enter code (コードを入力)]リンクをクリックします。

    IT部門で実装された機能に基づき、Oktaで保護されたアプリにアクセスする際、Okta Verifyから送信されたコードを入力してサインインするよう求められる場合があります。

  3. デバイスでOkta Verifyを開きます。
  4. 画面に表示される認証コードを覚えておいてください。Okta VerifyでTouch IDまたはFace ID が有効になっている場合、顔または指紋をスキャンしてコードを確認するよう求められる場合があります。
    6桁コードは30秒ごとに更新されます。進行状況バーに新しいコードが生成されるまでの残り時間が表示されます。
    Okta Verifyに複数のアカウントがある場合は、アクセスする組織に関連付けられているアカウントを使用します。
    アクセスしたいアプリがOkta Verifyと同じデバイスにある場合、コードをタップしてコピーし、[Enter a code (コードを入力)]ウィンドウに貼り付けます。
  5. [Enter a code (コードを入力)]ウィンドウでコードを入力し、[Verify (確認する)]をクリックします。

プッシュ通知を使用してサインインする

  1. コンピューターのウェブブラウザーで、組織のOktaアカウントにサインインするか、Oktaで保護されたアプリにアクセスします。
    [Okta Verify Authentication (Okta Verify認証)] チャレンジ画面が表示されます。
  2. デバイスにプッシュ通知を送信して本人確認を行います。[Send Push (プッシュ通知を送信する)]をクリックします。

    注意: このオプションは、IT 部門が有効にしていなければ利用できません。
    オプションとして、[Send push automatically (プッシュ通知を自動送信する)]チェックボックスにをオンにし、今後の通知がデバイス上に自動でポップアップされるようにします。
    IT部門が実装した機能に基づき、Oktaで保護されたアプリにアクセスする際、プッシュ通知を取得して本人確認を行うよう求められる場合があります。[Select (選択)]をクリックします。

  3. Okta Verifyからのプッシュ通知が届いているかモバイルデバイスを確認します。5分以内に認証リクエストを承認または拒否しない場合、通知は期限切れとなります。
    モバイルデバイスで新しい通知を受信するには、アクセスしたいアプリに戻り、もう1度プッシュ通知を選択します。
  4. アプリへのサインインを試みたことを確認するオプションをタップします。
    通知からリクエストを承認する方法は2つあります。
    • ロック画面では以下の操作を行います。
      1. 通知が表示されたら左にスワイプします。
      2. [View (表示する)]をタップします。

      3. [Yes, It's Me (はい、本人です)]または[No, It's Not Me (いいえ、本人ではありません)]をタップします。デバイスのロックを解除するように求められます。

      4. 先に進むには、PINを入力するか、指紋でデバイスのロックを解除します。
    • ロック解除画面ではプッシュ通知から以下の操作を行います。
      1. 通知が表示されたら下にスワイプします。
      2. リクエストのアクションに対して[Yes, It's Me (はい、本人です)]または[No, It's Not Me (いいえ、本人ではありません)]をタップします。
  5. Okta VerifyアプリでTouch IDま たはFace ID が有効になっている場合、顔または指紋をスキャンするよう求められます。


    10秒以内に顔または指紋をスキャンしない場合、Face IDまたはTouch ID をもう1度プロンプトされます。
    認証に失敗した場合、組織のOktaアカウントにサインインするよう求められます。電子メールアドレスを入力します。
    本人確認が完了したら、アプリのアカウントにアクセスできます。

デバイスを使用してサインインする

組織のIT部門がこの機能をサポートしており、Okta Verifyがデバイスを使用してサインインするようセットアップされている場合、アプリを開く際に[Sign in using Okta Verify on this device (このデバイスでOkta Verifyを使用してサインインする)]ボタンをクリックできます。

メッセージが表示されたら、Okta Verifyのプッシュ通知で[Yes (はい)]をタップし、本人確認を行います。

メッセージが表示される場合、Face IDまたはTouch IDを使用します。

サインインの確認メッセージが表示された後、アクセスしたいアプリに戻ります。

サインインの方法としてデバイスがセットアップされている場合、パスワードを入力せずにアプリのアカウントに安全にアクセスできます。アプリを開く際、Okta Verifyはバッググランドで実行し、サインインを行います。Okta Verifyで追加認証要素としてTouch ID またはFace IDが有効になっている場合、顔または指紋をスキャンするよう求められます。本人確認が完了したら、アプリのアカウントにアクセスできます。

Apple Watchで認証する

Apple WatchのOkta Verifyを使用して、ウォッチの画面から認証通知を表示したり、承認または拒否したりできます。

ウォッチとiPhoneをペアリングすると、Okta Verifyアプリが自動的にインストールされます。Appleのデザインに基づき、アプリをいつでも開いたり使用したりできますが、iPhoneがロック解除されている間はウォッチに通知が表示されません。

リクエストを許可した後、Oktaで保護されたアカウントに自動的にサインインします。

通知アクションで認証する

Okta Verifyプッシュ通知を長押しするか、下にスワイプして、Okta Verifyアプリを開かずに認証リクエストを承認または拒否できます。
