デバイスからOkta Verifyを削除してしまいました。IT 管理者に連絡せずに自分でOkta Verifyを再インストールしてセットアップできますか？

いいえ。現在のOkta Verify登録はお使いのデバイスから削除された特定のアプリにまだOkta Verify関連付けられています。再インストールしてOkta Verifyもう一度登録しようとしても、現在の登録がアカウントからOkta Verify削除されるまではOkta完了できません。この場合、現在の登録を削除できるのは IT 管理者Okta Verifyのみです。会社の IT 部門に連絡し、新しいデバイスを登録してもらってください。

デバイスを工場出荷設定に戻したら、アカウントにサインインできなくOktaなりました。再度バーコードをスキャンできますか？

いいえ。現在のOkta Verify登録は、まだデバイスを工場出荷値に戻したときに消去された特定のアプリに関連付けられています。Okta Verify 再インストールしてOkta Verifyもう一度登録しようとしても、現在の登録がOkta Verify削除されるまでは完了できません。この場合、現在のOkta Verify登録を削除し、新規デバイスを登録できるのは IT 管理者だけです。

新しいデバイスを登録するステップに従いましたが、手順の最後で緑色のチェックマークが表示されず、デバイスではエラーが発生します。何が問題なのでしょうか？

デバイスの時計が、携帯電話のプロバイダーのネットワークが維持する時間と同期していない可能性があります。デバイスの時計とネットワークの時計が少しずれているだけでも、登録に失敗する可能性があります。デバイスの自動日時設定をご確認ください。

再インストール後Okta Verify、プッシュ通知は受信しますがOkta Verify、それを承認しても何も起きません。なぜでしょうか？

この問題で考えられる解決法は以下のとおりです。

  • デバイスの時計が、携帯電話のプロバイダーのネットワークが維持する時間と同期していない可能性があります。デバイスの時計とネットワークの時計が少しずれているだけでも、登録に失敗する可能性があります。デバイスを確認し、自動日時設定が有効になっていることをご確認ください。
  • バックアップからデバイスを復元した場合は、Okta Verifyお使いのデバイスまたは新しいデバイスでの Okta Verify のリセット現在のデバイスまたは新しいデバイスでOkta Verifyをリセットする。
6 桁のパスコードを入力しても認証できなくなりました。Okta Verify 何が問題なのでしょうか？

この場合、考えられる理由は次のとおりです。

  • デバイスの時計が、携帯電話のプロバイダーのネットワークが維持する時間と同期していない可能性があります。デバイスの時計とネットワークの時計が少しずれているだけでも、登録に失敗する可能性があります。デバイスの自動日時設定をご確認ください。
  • バックアップからデバイスを復元した場合は、Okta Verifyお使いのデバイスまたは新しいデバイスでの Okta Verify のリセット現在のデバイスまたは新しいデバイスでOkta Verifyをリセットする。
バックアップからiOS のデバイスデータを復元した後、Okta Verifyアカウントはアプリに表示されますが、動作しません。

iOSデバイスでOkta Verifyをセットアップし、iCloudまたはmacOSでデバイスデータをバックアップしてから別のデバイスにこのバックアップのデータを復元する場合、Okta Verifyアカウントはアプリに表示されても動作していない場合があります。アカウントを削除してからもう1度追加します。Okta End-User Dashboardで[Settings (設定)] > Extra Verification (追加認証)]を開き、動作していないアカウントを削除します。デバイスでOkta Verifyを開き、削除するアカウントをタップして、[Delete (削除)]をタップします。アカウントをもう1度追加するには、「iOSデバイスでOkta Verifyのアカウントを管理する」をご覧ください。ヘルプが必要な場合、IT部門にお問い合わせください。

バックアップからiOS のデバイスデータを復元した後、Okta Verifyアカウントは最初に構成したサインイン方法を保持しません。

デバイスでOkta Verifyをセットアップし、デバイスデータをiCloudまたはmacOSでバックアップしてからデバイスにバックアップを復元する場合、Okta Verifyアカウントは最初に構成したサインイン方法を保持しない場合があります。[Account Details (アカウントの詳細)]ページで、すべてのサインイン方法が有効化されているように見える場合がありますが、認証コードのみが機能します。 この問題を解決するには、アカウントを削除してからもう1度追加します。アカウントを削除するには、Okta End-User Dashboardにアクセスします。[Settings (設定)] > [Extra Verification (追加認証)]を開き、Okta Verifyを削除します。デバイスでOkta Verifyを開き、削除するアカウントをタップして、[Delete (削除)]をタップします。ヘルプが必要な場合、IT部門にお問い合わせください。Okta Verifyをセットアップするには、「iOS搭載デバイスでOkta Verifyをセットアップする」をご覧ください。

Oktaエンドユーザーダッシュボードの [Settings (設定)] > [Extra Verification (追加認証)]には2つのOkta Verifyインスタンスが表示されます。1つ目はOkta Verifyと呼ばれています。両方のアイテムが必要ですか？

場合によっては 、Okta End User Dashboardの[Settings (設定)] > [Extra Verification (追加認証)] セクションで、1つではなく2つのOkta Verify登録がOkta Verifyの構成に含まれ、混乱が生じる場合があります。Okta Verifyと呼ばれる古いインスタンスを削除します。ヘルプが必要な場合、IT部門にお問い合わせください。

すでにアクティブなOkta Verifyアカウントがありますが、新しいデバイスに追加できません。

Oktaアカウントが一時停止になり、管理者により再アクティブ化された場合、既存のOkta Verifyアプリが動作しない場合があり、別のデバイスにこのアカウントを追加できません。この問題を解決するには、管理者にお問い合わせください。
