Service Users

Service users enable you to grant permissions to your automation to access specific operations in the Advanced Server Access Platform, such as:

  1. Enabling trusted services to be granted access to your infrastructure
  2. Integrating with the Advanced Server Access API to automatically add users to groupsGroups allow you to organize your end users and the apps they can access. Assigning apps to large sets of end users is made easier with groups.
  3. Retrieving audit events from the Advanced Server Access API to store in your ELK setup
  4. Building other custom integrations with the Advanced Server Access API

Service users can be added to a group just like a normal user, and they will be granted permissions the same way. Currently the main difference between a user and service user is how each type of user authenticates.

While users are pushed from Okta, a service user is given a pair of credentials that are used to generate a short-lived authentication token to be used with the API.

Authentication

In order to authenticate as your service user to the Advanced Server Access API, you will need to create an API key. The API key is a pair of strings known as the id and secret. You will need both to authenticate, which generates an authentication token that is sent with each request you make.

Create a Service User and API Key

  1. Navigate to the Users page for your team.

  2. Select the Service Users tab to see a list of your Service Users.

  3. Click the Create Service User button at the top of the list to begin creating your Service User.

  4. In the Username field, create a name for your Service User and press the Create Service User button.

  5. Once your user is created, click on the Create an API Key button and a screen will list your API Key ID and your API Key Secret. You will not be able to see your API Key Secret again, so copy and store it safely.

  6. To expire your Service Users API Key and ID, click the gear icon next to your existing API Key and press Expire Now

Note: Expiring an API key immediately will prevent any requests using a token generated with it from succeeding.
