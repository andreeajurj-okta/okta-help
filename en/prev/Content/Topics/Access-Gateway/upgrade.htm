Upgrade Access Gateway

Overview

The purpose of this tutorial is to walk through the process of upgrading Okta Access Gateway.

Prerequisites

  • Access Gateway has been backed up.
    The most common way to back up access gateway is by using the snapshot feature of your VM hosting environment.
    Care should be taken to always backup Access Gateway before making any system level changes.

  • You have command line access to Access Gateway

  • Access Gateway, is up and connected to the internet so it may download YUM updates.

Upgrading Access Gateway

To update Access Gateway:

Caution

During the upgrade process, Access Gateway will be offline and unavailable for between 3 and 5 minutes. Plan accordingly.

Note

To perform an upgrade from 2019.2.0 to 2019.4.2:
  1. Login to management console
  2. Select the System menu
  3. Select Remove Package
  4. .Remove oracle-instantclient18.3-basic . .
  5. .Return to the System menu and select Update
  6. Proceed with update normally.

  1. Using SSH connect to the Access Gateway command line console.

  2. Enter 5 to enter the System sub-menu.

  3. Enter 4 to enter the Update sub-menu.

  4. Enter one of the Upgrade sub-menu commands:
    1. 1 - View - Display a list of available updates
    2. 2 - Update
    3. x - Exit.
Viewing Upgrade Contents
  1. Enter 1 to view the elements of the upgrade.
    The system will display a scrollable list of all packages which are part of the upgrade.
    The listing will resemble:
    UP/DOWN/HOME/END - scroll list   x - exit
okta-access-gateway.x86_64			2019.1.2.20190924.12345-00001
okta-access-gateway-core.x86_64		2019.1.2.20190924.12345-00001
okta-access-gateway-core-apps.x86_64		2019.1.2.20190924.12345-00001
okta-access-gateway-core-events.x86_64		2019.1.2.20190924.12345-00001
. . .
  2. Use the UP/DOWN/HOME and END keys to move around the listing.
    Enter x to exit and return to the upgrade menu.
Performing Upgrade
  1. Enter 2 begin the upgrade.
    The system will display a list of update information as the update runs.
    The listing will resemble:
    Update in progress
. . .

Updates

. . . 
Transaction Summary
============================================================================
Install	2 Packages
Upgrade	28 Packages

Total download size:	152M
Downloading packages
. . . 
Update Successful.

    When complete the upgrade will display Upgrade Successful.

  2. Hit any key to complete the upgrade and return to the System menu.

    Note

    The upgrade report screen will then be displayed.
    Examine the yum log and transaction summary report is for upgrade information and details.
