Concepts

Auth Modules:

  • Represent a trust relationship between Access Gateway and one of the supported authorization types
  • Can by backed by multiple sources including: Okta, LDAP, Active Directory, Datastore and others.
  • Support combinations of authorizations using Authentication Module Chooser.
  • May be added, modified, deleted and otherwise managed within the Access Gateway UI.
Adding Auth Modules

To add an Auth Module:

  1. Navigate to your Access Gateway InstanceAn instance, or computer instance, is a virtual machine (VM) or individual physical computer, used to host a software appliance..
  2. Select the Settings tab.
  3. Select the Auth Module pane.
  4. Select (+)> and one of:
    TypeDescription
    Active DirectoryWindows based Active Directory auth module. Requires URL, Port, and credentials and other information for configuration.
    Auth ChooserAllows for the selection and use or two or more existing auth modules.
    Requires two or more existing Auth Modules.
    Desktop SSOKerberosKerberos is a computer-network authentication protocol that works on the basis of tickets to allow nodes communicating over a non-secure network to prove their identity to one another in a secure manner. based Auth Module.
    Requires URL, Port, and credentials and other information for configuration.

    LDAPLDAP based Auth Module.
    Requires LDAP URL, Port, Credentials and other mapping information.
    Local AuthLocal Auth Module, provided by Access Gateway itself. Requires an internal set of users and user mapping.

    OKTA

    Okta based Auth Module.
    Requires a Okta tenant which a pre-establised trust relationship with the Access Gateway Instance

    SAML

    SAML based Auth Module
    Requires access to local IDPAn acronym for Identity Provider. It is a service that manages end user accounts analogous to user directories such as LDAP and Active Directory, and can send SAML responses to SPs to authenticate end users. Within this scenario, the IdP is Okta. DomainA domain is an attribute of an Okta organization.     DataStore

    DataStore base auth module. Requires a pre-existing datastore, as well as mapping information.

After selecting Active Directory the Add New Auth Module page will be displayed, configured for Active Directory.

Configuring a Okta Auth Module

Note

An Okta Auth Module requires a defined trust relationship between the Access Gateway instance and the Okta tenant.

Define a trust relationship within Okta with the Access Gateway Instance

  1. Login to your Okta tenant and click the AdminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button on the upper right side of the My Applications page. button to become admin.
  2. In the Service Account Dashboard, select Security → API from the menu options.
  3. Select the Trusted Origins tab.
  4. Click Add Origin.
  5. In the Add Origin dialog specify:

    Field

    Value

    Name

    Meaningful name, such as Trusted Origin for Access Gateway.

    Origin URL

    URL for Access Gateway instance, such as https://gateway.tld

    Type

    Redirect.

Return to the Access Gateway UI.
