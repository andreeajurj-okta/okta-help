Setup Access Gateway using Oracle VirtualBox

Overview

The purpose of this guide is to walk you through the process of installing the Access Gateway virtual appliance using Oracle VirtualBox.

For a complete list of all supported installation targets see Setup Access Gateway Using an OVA Image

Prerequisites

  • You have reviewed all Prerequisites for Deploying Access Gateway

What’s covered in this guide

  1. Download the latest OVA image.
  2. Import to Oracle VirtualBox

Download the latest OVA image.

To download the latest Okta Access Gateway OVA Image

  1. Download the Okta Access Gateway image from the Settings > Downloads page in Okta.

  2. When prompted save the file to an appropriate location.

Import to Oracle VirtualBox

To Import to Oracle VirtualBox:

  1. Open Oracle VirtualBox.

  2. Select File > Import Appliance.

  3. Click the folder icon to browse to the directory where the OVA file is stored.
    Select the Okta Access Gateway OVA file, and click Open.

  4. Click Next.

  5. Specify Name "Access Gateway" and and verify Guest OS Type is set to Red Hat (64-bit).

  6. Select the Reinitialize the MAC address of all network cards option, and click Import.

  7. VirtualBox will begin the import process.

  8. Click Start to start the VM.

Next Steps

After Okta Access Gateway has been installed there are a number of common post installation tasks that should be performed.
Including:

  • Determine the IP Address of the Okta Access Gateway virtual application.
  • Configure admin /etc/hosts entry
  • Initialize the Command Line console
  • Reset Passwords
  • Initialize Access Gateway Admin UI Console
  • Configure DNS
  • Add Applications

For More Information

  • See Okta Access Gateway Post Installation for a complete task list.
  • See Access Gateway Command Line Management Console Reference for a complete list of management console commands.
  • See Access Gateway Admin UI Console Overview for an overview of the Access Gateway UI console.
