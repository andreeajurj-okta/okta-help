Managing Enrolled Devices

You can manage enrolled devices, as follows.

Viewing Devices

The Devices page displays all the devices in your org that have at one time been enrolled in Okta Mobility Management. Attributes displayed about each device include OS versions and the username and email address of the owner of each device, see below.

By default, all devices are displayed, however you can filter as follows:

  • Use the left pane filters to filter the device display results by: OS, Status, Device Activity, or by Device Security.
  • Use the search text box to search by user attributes (username, email, first name, and last name) as well as device attributes.
Deprovisioning

You can deprovision devices, perform a complete factory reset, reset/clear end users' passcodes, and remotely lock devices. You can also monitor all devices to see if any have been jailbroken or rooted. See below for details.

Important: OMM enrolled devices are automatically deprovisioned when a user is deactivated.

Deprovisioning and Factory Reset By User

Deprovisioning removes all managed apps and configuration and unenrolls the device from Okta Mobility Management.

Note: Deprovisioning does not require end-user confirmation. After you deprovision users, their managed apps and configuration are removed. They see the effects of the deprovisioning the next time they sign in.

To perform deprovisioning or factory resets by user:

  • For Android devices, the Admin resets the passcode.
  • For iOS devices the Admin clears the passcode and the end user needs to set up a new one themselves within one hour.

Note that this feature has a minimum requirement of Okta Mobile 1.2.8 for Android or Okta 4.2 for iOS.

To reset or clear a device passcode (depending upon device type) remotely:

  1. Select the Device Attributes page of the device whose password you want to clear or reset.
  2. Select Reset Passcode (for Android) or Clear Passcode (for iOS) (the button label changes according to device type).

Android:

For Android devices, enter the new passcode you want to assign to this device, then click Reset Passcode:

iOS:

For iOS devices, click Clear Passcode to confirm. The end user then has one hour to set a new passcode.

Jailbreak/Root Detection

Okta provides Jailbreak/Root detection for OMM devices.

About Jailbreak/Root Detection:

  • Jailbreak Detection: If an iOS device has been jailbroken, it means your end user has installed unsupported software on their device to either run custom software without Apple's permission, or to pirate copies of software or in app purchases.
  • Root Detection: If an Android device has been rooted, while it doesn't mean the end user has necessarily done anything inappropriate with their device, it does mean they have access to alter the firmware and/or OS that Google and the manufacturer/carrier shipped with the device.

Admins can quickly see if a device has been jailbroken or rooted by navigating to Directory > Devices from the Admin Dashboard to see a list of enrolled devices. Jailbroken/rooted devices are called out by an icon and label, as shown below:
