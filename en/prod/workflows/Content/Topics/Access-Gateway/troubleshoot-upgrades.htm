Troubleshooting upgrades

Common problems and solutions for troubleshooting the Access Gateway upgrade process.

Topics

  • Upgrading from 2019.2.0
  • Error accessing yum.oag.okta.com

Upgrading from 2019.2.0

When updating from Access Gateway 2019.2.0 you must first remove the package oracle-instantclient18.3-basic.

To remove the package:

  1. Login to Access Gateway Management console.
  2. Select the System menu.
  3. Select Remove Package.
  4. Remove oracle-instantclient18.3-basic .
  5. Return to the System menu and select Update.
  6. Proceed with update normally.

Error accessing yum.oag.okta.com

If the Access Gateway yum repository is unreachable, you will receive an upgrade error resembling:

Ensure that the repository, yum.oag.okta.com, is reachable from the Access Gateway node being upgraded. You can test access using a variety of commands.

Using the Access Gateway Management console, under Network choose d and then enter yum.oag.okta.com.
For example:

Access Gateway Administration ...

Using traceroute ensure that their is a route to yum.oag.okta.com from inside the network hosting Access Gateway.
Note that traceroute will NOT be able to trace a complete route to yum.oag.okta.com due to limitations of AWS EC2 configuration.
For example:
