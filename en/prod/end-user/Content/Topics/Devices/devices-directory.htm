Device directory

This is an Early Access feature. To enable it, contact Okta Support.

You can view your organization's Device Trust-enrolled Windows or macOS inventory from the Admin Console. Your device inventory is stored in Universal Directory. It allows you to create and enforce granular device access policies to enable end users to seamlessly enroll into the Okta device identity service and access protected resources.

Procedures

Add a device

You add devices using the Create Devices API. For details, see ......

 

View devices


View all the devices in your org.

Go to Directory > Devices. From here you can do the following:

  • Filter by platform and/or access status
  • Activate created and deactivated devices
  • Deactivate suspended devices
  • Suspend active devices. Suspended devices are blocked from accessing device trust-protected resources.
  • Unsuspend suspended devices
  • Delete deactivated devices


View all the devices linked to a specific user:

  1. Go to Directory > People.
  2. Click a user.
  3. On the user's page, click the Devices tab. All the devices linked to the user are listed.

    4. Revoke access is the only action you can perform on a device from a user's Devices tab. This revokes client certificates, which in turn disassociates the user from the device and blocks the user's access to device trust-protected resources.


View all the users linked to a specific device:

Note

During the Early Access phase of this feature, only the first 20 users linked to a device are shown.

  1. Go to Directory > Devices.
  2. Click a device to which one or more users are linked.

    3. The Device users table lists the first 20 users linked to the device (EA). The table provides the following:

    • Enrollment date – The date the device was enrolled in Okta Device Trust and a certificate was issued.
    • Available actions – Revoke access (if Active), Suspend or Deactivate (if Active), Unsuspend (if Suspended), Activate or Delete (if Deactivated)

Device lifecycles

Status Description
Created

Created devices exist in UD but have not yet been enrolled by a user. This status is typical if your organization created records for devices prior to assigning them to users. Enrolling a device changes its lifecycle from Created to Active.

To create a device record, [include a link here to API docs ... ]

Created devices:

  • Cannot be used to access protected resources
  • Are not enrolled/assigned to users
Active

Active devices are linked to at least one user. A device must be Active to be evaluated by your app sign-on policy.

When a device is Active:

  • You can link users to the device
  • Enrolled users can access protected resources if permitted by the app sign-on policy

When an Active device is Suspended:

  • All users assigned to the device are denied access to any protected resources that were accessible prior to suspension.
  • Device certificates are unaffected
  • User assignment(s) remain

When an Active device is Deactivated:

  • Users are denied access to any protected resources that were accessible prior to deactivation.
  • Device certificates are revoked
  • User enrollment(s)/assignment(s) are removed
Deactivated

Active and Suspended devices can be Deactivated.

When a device is Deactivated:

  • Device certificates are revoked
  • All users assigned to the device are denied access to any protected resources that were accessible prior to deactivation
  • User enrollment(s)/assignment(s) are removed
  • Existing user session is unaffected; user continues to have access to protected resources until the session ends

When a Deactivated device is Activated:

  • The device must be re-enrolled before a user can establish an Okta session from the device
  • Once the device is enrolled/assigned, users can access device trust-secured apps if permitted by the app sign-on policy

Suspended

Only Active devices can be Suspended.

Suspension is useful if you need to pause and later resume device access for users such as contractors or employees who take a leave of absence.

When a device is Suspended:

  • All users assigned to the device are denied access to protected resources that were accessible prior to suspension
  • Device certificates are unaffected
  • New enrollments are permitted
  • Existing user enrollment(s)/assignment(s) remain

You can perform the following additional actions on devices in your inventory:

Action Description
Delete

Available only on the Devices page and Device Profiles page for Deactivated devices.

Deleted devices:

  • Are deleted from Universal Directory
  • No longer appear in the Admin Console nor the API
Revoke access

Available only from the Devices tab of user profile pages.

Revoking access blocks a single user's access to protected resources that were accessible prior to revocation.
Unsuspend

Available only from the Device Profile page of a specific device.

Click to unsuspend a suspended device. Unsuspending a device restores access to protected resources for all users assigned to the device.
