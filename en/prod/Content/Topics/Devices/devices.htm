About Okta Mobility Management

  • The OMM menu is only available to orgs that implement Okta Mobility Management (OMM).
  • Procedures documented on this page are only available to customers who have already purchased OMM for their organization. New OMM sales are not supported. For more information, contact Okta Support.

The Okta Mobility Management page displays all the mobile devices that are enrolled in your org. It provides an overview of device enrollment status, security status, operating system (OS) type, and 30-day activity. It also includes a reporting option that allows you to download a CSV file.


View device attributes

The Device Attributes page displays device-specific information. Okta polls devices once every 24 hours to update this information. If a device is offline when Okta polls, the device is immediately polled when it is back online. Be aware that the device must be online (cellular or WiFi) for hardware and software specifications, and the device must be online (cellular or WiFi) and unlocked to view the security status.

To view the Device Attributes page for a device, complete one of the following:

  • In the Admin Console, go to Directory > People. Click a user in the Person & username column, click the Devices tab, and then click a device in the Device name column.
  • In the Admin Console, go to Directory > Devices, and then click a device in the Device Name & User column.
  • In the Admin Console, go to OMM > Okta Mobility Management, and then click a device in the Device Name & User column.

Wipe data from a device

Click: Important to know before you begin

After you select a wipe option, data wiping occurs as follows:

  • Devices are wiped and unenrolled from OMM as and when they come online (with either a cellular or WiFi connection).

  • Offline devices are not wiped until they come back online. Until then, end users will continue to have access to any corporate data on the device. Such a device may still show as Enrolled in the Status column of the DevicesOMM page until it comes back online.

To help you protect the sensitive corporate data that your end users and contractors access from OMM-enrolled devices, the Devices page provides two data wipe options:

  • Wipe only company data from a device

This option removes only managed apps and configurations and unenrolls the device from OMM. Once wiped, these devices are listed as Deprovisioned in the Status column of the upper dashboard and in the downloadable CSV report. When an end user is deactivated from Okta, their OMM-enrolled device is automatically deprovisioned and all company-managed apps and data are removed from the device in OMM. If the user is reactivated later, they must re-enroll their device. Wiping company data does not require end-user confirmation. After you wipe a device of company data, end users see the effects of the wipe the next time they sign in. Best practice

Before you unenroll an end user from OMM, first wipe company data as described in this procedure. Additionally, you may want to discourage end users from removing or unenrolling from OMM themselves; otherwise, the Okta servers may never detect the removal (for example, if there is a poor network connection or if the device is offline). This creates a scenario in which a deprovisioned device is still shown as Enrolled on the DevicesOMM page. This scenario can occur regardless of a device's rooted or jailbroken status.

  1. Choose a way to access this wipe option:

    1. In the Admin Console, go to Directory > People.
    2. Click the user whose device you want to wipe of company data.
    3. Click the DevicesOMM tab.
    4. In the Device Name column, locate the device you want to wipe.
    5. Proceed to Step 2.

    – or –

    1. In the Admin Console, go to OMM > Okta Mobility Management.
    2. In the Device Name & User column, locate the device you want to wipe.
    3. Proceed to Step 2.

  2. In the Actions column, click the   icon, and then click Wipe Company Data.
    Okta-managed native apps are wiped. Personal apps, content, and settings are not wiped. (You can also do this from the Device Attributes page using the Device Actions drop-down menu.)

Note: When an end user is deleted from Okta, their device no longer appears in the device report. For details about generating reports, see Reporting below.


  • Wipe all data from a device

This option removes all apps and data from the device and returns the device to its factory settings. Like the Wipe Company Data option, this option also unenrolls the device from OMM. Once wiped, these devices are listed as Deprovisioned under the Status column and the device appears in the device report as Deprovisioned. If the user is reactivated later, they must re-enroll their device.

Detect Jailbroken/Rooted devices

Okta provides Jailbreak (iOS) and Root (Android) detection for OMM devices.

  • Jailbreak Detection: If an iOS device has been jailbroken, it means your end user has installed unsupported software on their device to either run custom software without Apple's permission, or to pirate copies of software or in-app purchases.
  • Root Detection: If an Android device has been rooted, while it doesn't mean the end user has necessarily done anything inappropriate with their device, it does mean they have access to alter the firmware and/or OS that Google and the manufacturer/carrier shipped with the device.

You can determine from the DevicesOMM page whether a device has been jailbroken or rooted. Affected devices are indicated by an icon and label, as shown below:

Clear or reset device passcodes

When passwords are lost or need to be reset for any reason:

  • For Android devices, the admin resets the passcode.
  • For iOS devices, the admin clears the passcode and the end user must set up a new one themselves within one hour.

Notes:

  • This option has a minimum requirement of Okta Mobile 1.2.8 for Android or Okta Mobile 4.2 for iOS.
  • iOS devices that have been rebooted or reset may not receive the Clear Passcode command if they do not have mobile data connections (3G, 4G, or LTE), as they do not automatically join WiFi connections until a passcode is entered. For more information, see this article.
  1. Go to Devices > Devices.

  2. Go to OMM > Okta Mobility Management.

  3. Click the device whose password you want to clear or reset.
  4. On the Device Attributes page, click Device Actions on the upper-right side of the page.
  5. Click Clear Passcode (for iOS) or Reset Passcode (for Android).

iOS

For iOS devices, click Clear Passcode to confirm. From that point on, the end user has one hour to set a new passcode.

Android

For Android devices, enter the new passcode you want to assign to this device, then click Reset Passcode.

Important: If you are resetting the passcode of a Samsung SAFE or Native Android device, make sure that the passcode you enter complies with the General Android Device Passcode Requirements configured in the Platform Rule (OMM > OMM Policies). Be aware that, even though you are not prevented from entering a non-compliant passcode in the New Passcode field and a success message may display after you click Reset Passcode, the non-compliant passcode will not work and the end user will not be able to access apps.

Lock devices remotely

You can remotely lock a device. This option is useful when devices appear to be in an uncertain state but wiping (factory reset) is not warranted.

  1. Go to Devices > Overview.
  2. Go to OMM > Okta Mobility Management
  3. Click the device you want to reset.
  4. On the Device Attributes page, click Device Actions on the upper-right side of the page.
  5. Click Remote Lock.
  6. (Optional) Send a message and contact number to the device. For example, "Please return this phone to Vivek B" and a contact number.
  7. Click Remote Lock.

Generate a device report

To see granular data about each device, you can generate a report by clicking Download CSV. The resulting .csv file provides comprehensive per-device information, including device status, serial number, platform, device capacity (MB), and available device capacity (to name a few).

Android note: To provide users with greater data protection, starting in 6.0.x release, Android removes programmatic access to the device's local hardware identifier for apps using the Wi-Fi and Bluetooth APIs. For details, see this Android documentation.
