People

The People page is located under the Directory menu. This is where you can add and view the end users within your organization. You can quickly drill into individual end users to view their profiles, group memberships, and app assignments to take administrative action.

To find this page from the Administrator Dashboard, click Directory, then choose People.

Main page

  • Top buttons allow you to easily add people and reset passwords. The More Actions button allows you to import users from a CSV file , activate and deactivate people, and reset passwords for Okta-mastered users.
  • The alphabetical last name navigation bar provides quick access to the names you know.
  • The Filters side bar provides a count of key data information and process states, including the overall number of users, activated users, the number of users in pending activation, and the current number of deactivated users. It also provides the state of users requiring a password reset.
  • The default view shows up to 25 users, but the Show More button allows for continual scrolling for increased visibility of all users.

Adding People

Adding people to your organization allows them to have their own My Applications page. Do the following to add end users to your org.

  1. From the People page, click Add Person.

    The Add Person page is displayed.

  2. Enter the First name and Last name.

  3. Enter the Username.

    The username must be in email address format, and is typically the user's primary email address. Users sign in using this username.

  4. Enter a Primary email address.

    This can be any valid email address that the user has access to. Typically this is the same address used for the user's login name.

  5. Optionally, enter a Secondary email.

    This email can be used as a back up in case the user can't access their primary email. This is not required.

  6. Click Add Person, or add another by clicking Save and Add Another.

  7. If you click the Send user activation email now check box, your end user immediately receives their Welcome to Okta! activation email. Otherwise, these users are at Pending Activation status, and are not notified via email of their Okta account.

Activating People

Activating people in your organization changes their account status from Pending to Active. Active end users receive an email that steps them through the process of setting up their unique account within your org.

From the Okta People, page you can view the status of each of your end users from the Status column (the far left column on the page). Their status may be Pending Activation, Active, Password reset, or Deactivated.

There are two ways to activate end users, depending on when their accounts were created:

  • If the user already appears under your Person & Username list and they are pending activation, you can activate them simply by clicking the Activate link, found in the Status column directly across from their name.
  • To activate one or more people who have been recently added, do the following:
  1. From the More Actions menu, click Activate. The Activate People page appears.
  2. From the list of users, select the users you want to activate and click Activate Selected. Or, click Activate All to activate everyone that appears on the list.
  3. An Activate People page appears. Click Activate to activate the chosen end users, or click Cancel to abort. When activated, each user appears as Active in the Status column.

An email is sent to each user's primary or secondary email address, informing them that their accounts are active. Once active, they can access all the provisioned applications assigned to them.

If you reactivate a person who was previously deactivated, the user is re-imported, but their apps remain unassigned.

The Just In Time provisioning feature (JIT) enables automatic user account creation in Okta the first time a user authenticates.

Using and Enabling Just In Time Provisioning

Just In Time (JIT) provisioning enables automatic user account creation in Okta the first time a user authenticates with one of the following methods:

  • Active Directory (AD) Delegated Authentication
  • Desktop Single Sign On
  • Inbound SAML

Requirements

JIT only works for users who are not already Okta users. This means that users confirmed on the import results page, regardless of whether or not they were subsequently activated, are not eligible for JIT activation. When JIT is enabled, users do not receive activation emails.

If you are using JIT provisioning with AD users, they must be imported first. After you enable JIT, import user accounts from AD. The import process defines the set of AD accounts that can be used to create Okta accounts (whether via JIT or the confirmation process). AD accounts that are not on the import list cannot be used to create Okta accounts.

Enabling JIT

Perform the following steps:

  1. From your Administrator Dashboard, select Security > Authentication and select the JIT Provisioning tab.
  2. Click the Edit button and select the Enable Just In Time Provisioning check box.
  3. Click the Save button.

Suspending and Unsuspending People

Note: This is an Early Access feature; contact Okta Support to enable it.

Suspending people in your organization changes their account status from Active to Suspended. Suspended users cannot log in to Okta and receive a message indicating they are suspended. The Okta login is blocked for apps that support SAML. App and group memberships are maintained for suspended users and are reinstated if the user is subsequently unsuspended.

Unsuspending people changes their account status from Suspended to Active. All app and group memberships are restored.

Suspending people is useful for temporary and contract workers and employees on leaves of absence if you want to pause and later resume access. This state is also useful if an employee leaves but the admin wants to review group and app assignments.

To suspend an end user click the Suspend button on a user's page.

A list of all suspended users is available as a filter.

To unsuspend a user, click the Unsuspend button.

Deactivating People

Deactivating people in your organization changes their account status from Active to Deactivated. Deactivating end users revokes their app and group memberships.

To deactivate an end user:

  1. From the More Actions menu, click Deactivate. The Deactivate People page appears.

  2. From the list of users, select the users you want to deactivate and then click Deactivate Selected.

  3. A Deactivate People dialog appears. Click Deactivate to confirm, or click Cancel to abort.

A deactivation email is sent, and the end user is listed as Deactivated under the Status column.
Once deactivated, a user's profile cannot be changed unless it is again made active.

Using Tombstone to Reuse a Username

This is an Early Access (EA) feature; contact Okta Support to enable it.

The tombstone feature allows you to reuse a username that was similar to that of a deactivated user. If you attempt to add a new user with the same username as another user, you will receive the following error message:

If the common name is from a deactivated user, use tombstone to reuse the username.
To do so,

  1. From the People page, find the deactivated user from the list of users.
  2. Click the name of the deactivated user.
  3. From the deactivated user's page, choose the Profile tab.
  4. From the right-side panel, click the Tombstone Username button. A confirmation modal appears.
  5. Click the Tombstone Username button to confirm. (This action cannot be undone).
  6. You are now able to use the username for the new, active user.

Unlocking an End User's Account

Note: This is an Early Access feature which does not appear in Okta unless it has been enabled. To enable it, call Okta Support.

To unlock a locked user account:

  1. Go to Directory > People.

    Locked users are shown as Locked out in the Status column.

  2. Click the user whose account you want to unlock.
  3. In the Applications tab, click More Actions, and then click Unlock Account.

Resetting an End User's Password

To reset an end user's password, do the following:

  1. From the People page, click Reset Passwords.
  1. On the Reset Passwords page, select the users whose password you'd like to reset.
  1. Click Reset Password.

An Account Password Reset email is sent to the specified email address. It includes an auto-generated password.

An end user who has forgotten their password can also reset it themselves using SMS. With SMS configured, end users can have Okta send them a text message with a password reset code.

Note: This feature is free in the United States and Canada. Admins with international orgs should contact customer support before enabling SMS.

SMS Password Reset Procedure

Perform the following steps to configure SMS:

  1. From your Administrator Dashboard, select Security > General.
  2. In the Organization section, click Edit and select Enabled from the Allow SMS for self-service operations (such as reset password or unlock account) pull-down menu.
  3. Click Save.

The next time you log in, you are presented with a Please update your profile screen. Either click Add Phone Number to set up your phone for SMS messages, or click Remind me later (you will be reminded with the same screen after 30 days).

The SMS feature includes a SMS Usage category on the Reports page. The SMS Usage Report enables admins to monitor the number of SMS messages sent.

Setting Up SMS Password Reset – New Users

  1. When new users sign into Okta for the first time they can set up their phone for SMS password reset by clicking the Add phone number button.
  2. The user is asked to enter a mobile phone number. This enables an initial verification code to be sent to their phone.
  3. The user enters the verification code sent to their phone and is then authenticated into Okta.

Setting up Password Reset – Existing Users

Existing users can reset their SMS passwords from their Account page.

  1. Click on the Account tab of your home page and in the Forgot Password Text Message section, click Add phone number.
  2. The user is asked to enter a mobile phone number. This enables an initial verification code to be sent to their phone.
  3. The user enters the verification code sent to their phone and is then authenticated into Okta.

Resetting Your Password with SMS

If you forget your password, you can do the following:

  1. On the sign-in page, click the Forgot password? link.
  2. Click the Send Text Message button.

  3. Continue through the prompts to reset your password.

Resetting or Reconfiguring Your Phone

Users who lose a phone or get a new number can reset or reconfigure their phones on the Account tab of their homepage.

Expiring End Users' Passwords

To expire all Okta-mastered end-user passwords, do the following:

  1. From the People page, click More Actions > Expire Passwords.
  2. On the confirmation page, click Expire Passwords.

This expires all of your Okta-mastered end-user passwords but does not affect your AD- or LDAP-mastered end-user passwords. After you expire your end-user passwords, you can use the App Password Health Report on the Reports page to monitor how your users reset their passwords.

Password Reset for Okta Mastered Users – detail

You can expire passwords for specific Okta-mastered users using the Temporary Password feature or for every user with the bulk Expire Passwords on the People page on the Okta Dashboard. When users have expired passwords, they appear in the People page with the status Password expired.

Important: When changing your Okta password, all applications assigned to users supporting Provisioning, and that are Password Push enabled, are updated with the new password. If responding to a security vulnerability, ensure that your applications are already patched and no longer vulnerable before resetting the Okta password.

Bulk User Password Expiration

You can expire passwords for all Okta-mastered users with one click. Every Okta-mastered user will be forced to change their password on next login.

From the Okta Administrator Dashboard, select People. Then select the Expire Passwords button.

The following confirmation screen appears. Select Expire Passwords to confirm the bulk expiration.

  • Bulk password expiration only applies to Okta-managed users. The passwords for users managed through Active Directory and LDAP delegated authentication are not expired. Your Active Directory and LDAP agents will continue to work even if the service account managed by Okta has an expired password. For more information on resetting passwords in Active Directory, see Active Directory Password Reset.
  • Active sessions remain active. The user is prompted for a new password at the next Okta login.
  • API tokens are not expired. API tokens are valid for 30 days and renew automatically with each request to Okta. For more information on API token expiration and revocation, see API Tokens.

Individual Password Expiration

When viewing a user on the Okta Administrator Dashboard, you can reset the password for that user and assign a temporary password for the user, by selecting the Temporary Password button, as shown below. Be sure to distribute the new password to the user securely; for example, by email or voicemail. The next time the user signs into Okta, the user must enter the temporary password and create a new password.

The temporary password is displayed for your information.

Note: After you generate a temporary password, you cannot create a password reset link.The message Password reset. User is now in one-time password mode. is displayed when viewing the user.

Portal/External Users

If your Okta organization powers an external user portal, the bulk password expiration feature may not be a viable solution. To use bulk expiration, your portal must support a password expiration flow and handle the following error code for the Create Session API operation.

Error code: E0000064
Description: Password is expired and must be changed.
HTTP return code: 401

Caution: If you change the default password policy to expire passwords or use the bulk password expiration feature, your application must handle this error.

Okta API

The Okta API provides a credential lifecycle operation to expire a password for a specific user. The API provides the flexibility to expire only the current password without generating a new temporary password.

 

Adding Applications

When setting up or maintaining end users, you can assign the applications you want to display on your user's My Applications or "Home" page. Assigning these applications can be done from the People page and/or the Applications page.

To add applications from the People page:

  1. Go to Directory > People.
  2. Click an end user's name.
  3. Select the Applications tab.
  4. Click Assign Applications on the Assigned Applications screen.
  5. You can select the desired application from the of available applications or use the Search box to search for the desired application by name. Once you have located the application you want to assign, click Assign App.
  6. Enter login information such as the username assigned to the person for the application. This is not the Okta login username, but the username the person uses for logging in to the application individually.

    Select application data options as needed. Each application has its own list of required application data. This application data can be additional parameters that define the user to the application. For example, some applications require profile or roles. You will also have to select credential security options.

  7. Click Save.

For details on how to assign applications from the Applications page, see Applications.

Note: You can assign applications to people manually, person by person, as described above. However, because this does not scale well, we recommend that you assign applications based on groups. For details, see Importing and Using Groups in Okta.

Removing Applications

You can unassign a user from one or more applications, thereby removing the app from the user's My Applications or home page.

  1. Go to the People page.

  2. Click the end user's name.

  3. From under the Assigned Applications banner, click the X (under Actions) to remove the user's access to that application. This removes the application button from the user's My Applications page.

Note: If the application assignment occurs within a Group of which the user belongs, the application must be unassigned from the entire Group.

Managing Individual People (End users)

Admins can manage users in groups or individually. Each end user has a unique identity account within Okta, from which the admin can mange their applications, group memberships, and general account information.

To manage an individual end user, simply go to Directory > People and click on an end user's name. Choose one of the following tabs:

Applications tab

This page lists all of the end user's current application assignments. Click Assign Applications to add more. To do so:

  1. Go to People > Applications tab.
  2. Click Assign Applications under the Assigned Applications banner.
  3. From the list of available applications, select the desired application.
  4. Select application data options as needed.

    Each application has its own list of required application data. This application data can be additional parameters that define the user to the application. For example, some applications require profile or roles. For these types of applications, you also have to select credentials security options from the drop-down menu.

  5. Enter the username assigned to the person for the application. This is not the Okta login username, but the username the person uses for logging in to the application individually.
  6. Click Save.

Groups tab

This page displays an end user's current group memberships, and allows an admin to add or remove them from a group.

Note: if you wish to add or remove more than one user (or "bulk" users), see Groups Page, below.

Adding an end user to a group

  1. Go to People > Groups tab.
  2. Enter a valid group name into the field under the Groups banner. If the group is valid, it will autocomplete your entry.

  3. Click Add. The end user is now a member of that group.

Removing an end user from a group

  1. Go to People > Groups tab.

  2. From the list of group names, click the x of the group from which you wish to remove the end user.

  3. The group is removed from the list and the end user is no longer a member. This does not delete the group itself.

Profile tab

You can edit a user's attributes using the Profile tab. This page shows all applicable attributes for a user: first and last name, username, primary and secondary email addresses, to name a few. To edit an attribute, do the following:

  1. Go to People > Profile tab.
  2. Click Edit.
  3. Enter the new attributes in the relevant fields as needed. A primary email must be listed.
  4. Click Save.
