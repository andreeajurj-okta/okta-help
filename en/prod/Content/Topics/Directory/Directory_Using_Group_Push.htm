Using Group Push

The Group Push feature allows administrators to use existing groupsGroups allow you to organize your end users and the apps they can access. Assigning apps to large sets of end users is made easier with groups. and group members in Okta, then push the group and maintain their memberships in Office 365, Google Apps, or Box.

Group Push cannot be used to create new groups—this can be done from within the People Page (see Using the Okta People Page) (People > Groups > Add Group button). Adding individual apps to a group is done within the Applications Page (see Using the Okta Applications Page) (Applications > appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. > Groups tab > Enter the group name in the add this app to a group field). For Group Push to work successfully, the group being "pushed" should include all of its required apps.

Note: You must have Provisioning enabled in each app that uses this feature.

Best Practice: Don't make changes in the target app. Make changes in Okta and then push the changes to the third-party app.

Prerequisite

In order for Group Push to work successfully,

  • If using Google Apps as the target app, groups using Group Push should have information specified in their Description field. This is generally done when the group is created. Google Apps also restricts the use of spaces in group names.
  • API access must be enabled in the target app.
  • For a user to be successfully pushed to an app through Group Push, they must already be assigned to that target app. This applies even if the group itself was not assigned to the app.
  • You must have provisioning enabled in the target app. If it is not enabled, you are prompted to enable it before proceeding, as shown below.

Using Group Push

See the following for details on using Group Push to push groups to a target app. Using Box as our example,

  1. Access your Okta instance of Box.

    Important: You must have full Box adminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button (shown below) on the upper right side of the My Applications page. credentials to push groups to Box. Accessing Box with co-admin credentials will cause the group push to fail.
  2. Within the app, choose the Push Groups tab.
  3. Click the green Push Groups button to add one or more groups. Groups can be added by name or by rule. Keep in mind that you cannot push to groups that already exist in the target app.

    4. If you keep the Push group memberships immediately default (which is checked), the selected group is immediately pushed to the target app.

  • Find groups by name provides a simple search field with autocomplete capability.
  • Find groups by rule is a helpful option when there are a large number of groups and/or a known naming convention for them. The wizard allows you to create a rule and specify its search criteria. Once created, the rule name is shown under the By rule filter and the found groups are listed under the Group in Okta column.

  1. Once populated with selected groups, use the Bulk Edit button to delete or deactivate the active groups. Simply select one or all of the groups from the list.
  • Delete button: permanently removes the group from Okta and the app (e.g., Box). A message appears with two options for deleting groups:

Delete the group in the target app — this option deletes the group and all its associated memberships.
Leave the group in the target app — this option tells Okta to stop pushing memberships, but the group remains in the target.

Note: You cannot delete a group that was picked up "by rule". In order to delete the group mapping, you must delete the rule that discovered the group.

  • Deactivate button: pauses the synchronization of groups, retaining their appearance in the app (e.g., Box). In this mode, you're able to keep adding new members within the app, but those changes won't appear in Okta.
  1. You can also delete and deactivate specific groups by clicking the status button to the right.

    The Push now option serves to "force" a push in the rare occurrence when the state of Okta and the target app are no longer in sync.

    The Information button (alongside the Active pull-down menu) displays creation information and group type. When an error occurs, it provides helpful troubleshooting information. See Troubleshooting below, for details.

Troubleshooting

Groups appearing in the target app without their users

If you have successfully pushed a group to the target app, but the assigned group members do not appear, verify that one of the following is true:

  • The target app has been added to the new group.
  • All group members of the new group are assigned to the target app (even if the group itself was not assigned to it).

If some group members are assigned to the target app and others are not, only successfully assigned members will appear in the target app.

Diagnosing errors

When an error occurs, Group Push provides information to diagnose the problem. The Pushed Groups panel includes an Errors column where errors are listed.

  • The red Information button displays the "time of failure" details and the probable cause.
  • The Retry All Groups button allows you to simultaneously re-push the groups after corrections are made.
