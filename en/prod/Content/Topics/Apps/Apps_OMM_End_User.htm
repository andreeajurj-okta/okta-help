Okta Mobility Management End-User Setup

Overview

The Okta Mobility Management program secures your mobile device and configures it to access your work email, calendar, contacts and applications. As part of the enrollment you are required to set a pin or password on your mobile device. The Okta policy allows for a 4-digit pin.

Your privacy is very important to Okta. Corporate data is protected while your personal data remains separate and not accessible to your organization. We only manage information required to secure and protect corporate data.

 

The following list shows what we can and cannot manage:

What OMM Can See What OMM Cannot See
Device Name Personal Mail, Calendars, and Contacts
*Phone Number SMS or Text Messages
Serial Number Browser History
Model Name and Number FaceTime or Phone Call Logs
Capacity and Space Available Personal Reminders and Notes
Capacity and Space Available Frequency of All Use
OS Version Number Device Location
**Names of Installed Apps Personal Reminders and Notes

*As derived from company directories, not from the phone.

**For personal apps, OMM can only see the names, none of the content of the apps.

The Okta Mobility management program has permission to deprovision only your work accounts and applications, which requires a special request with IT. You may factory reset your device erasing all of its contents if it is stolen. Note that your IT department can also factory reset your device.

The following procedures assume that you are currently an Okta user and assigned to a group that includes a mobile policy.

Setting Up Okta Mobility Management on your iOS Devices
  1. Install the Okta Mobile application on your iOS device.
  2. Tap the Okta Mobile app, then sign in to Okta Mobile:

  3. Set up and confirm a new Okta Mobile PIN:



    Okta recognizes that the device and Org are both registered and eligible for Okta Mobility Management (OMM) and automatically prompts you for enrollment.
  4. The Mobile Configuration Setup screen appears. Select Install Profile:

  5. The installation process begins. During the installation, the following iOS screens/prompts appear:

    Note that during the installation process you may be prompted for your device's pin.
    • The Install Profile screen. Select Install to continue.

    • A Warning screen. Select Install to continue.

    • A Remote Management popup dialog asks if you trust this profile's source. Select Trust:


  6. Once the installation is complete, you may need to select Done.

  7. The device is now enrolled in OMM and the Okta Apps Store is displayed.

    If you have been assigned any mobile-enabled applications, you will immediately see those apps listed in the Okta Apps Store. You still need to install each application.

  8. Install any app on your device by selecting INSTALL. Once an app is installed, the INSTALL button turns into an OPEN button. Select OPEN to configure the native app for this device.

Note: If any of the apps already exist on the device, they are considered unmanaged and are not subject to Okta's security policies, they will however, show up on the Okta Apps store page. You should remove any unmanaged versions of managed apps from your device, then reopen the Okta mobile app.

Note: If the Managed Open-In Policy is set to Deny access for managed apps to open in unmanaged apps, documents in managed apps can only be opened by other managed apps installed on that device.

Setting Up Okta Mobility Management on your Android Devices
  1. Install the Okta Mobile application on your Android device.
  2. Tap the Okta Mobile app, then sign in to Okta Mobile:

  3. Set up and confirm a new pin number. Select Done when complete.

  4. A Mobile Configuration Setup dialog appears. Select Enroll. Note that during the enrollment process you may be prompted for your device's pin.


  5. The The Device Administrator screen appears. Select Activate.

  6. The Okta Apps Store appears on your device:

  7. If you have been assigned any mobile-enabled applications, you will immediately see those apps listed in the Okta Apps Store. You still need to install each application.
  8. Install any app on your device by selecting INSTALL. Once an app is installed, the INSTALL button turns into an OPEN button. Select OPEN to configure the native app for this device.

Note: If any of the apps already exist on the device, they are considered unmanaged and are not subject to Okta's security policies, they will however, show up on the Okta Apps store page. You should to remove any unmanaged versions of managed apps from your device, then reopen the Okta mobile app.

Note: If the Managed Open-In Policy is set to Deny access for managed apps to open in unmanaged apps, documents in managed apps can only be opened by other managed apps installed on that device.

Adding New Apps to your Device

When a new app is made available for you, that app automatically appears in the Okta Apps Store on your device. Click INSTALL to install.

Note that you can uninstall a managed app at any time. Okta Mobility Management does not prevent them from doing so.
