Okta Extensions for iOS

This Extension allow users to perform SWAAn acronym for Secure Web Authentication. SWA is a SSO system developed by Okta to provide single sign-on for apps that don't support proprietary federated sign-on methods or SAML. Users can enter their credentials for these apps on their homepage. These credentials are stored such that users can access their apps without entering their credentials each time. When users first sign-in to a SWA app from their homepage, they see a pop-up message asking if they were able to sign-in successfully. and SAMLAn acronym for Security Assertion Markup Language, SAML is an XML-based standard for exchanging authentication and authorization data between an identity provider (IdP) and a service provider (SP). The SAML standard addresses issues unique to the single sign-on (SSO) solution, and defines three roles: the end user, the IDP, and the SP. functionality directly in Safari without having to open Okta Mobile.

Overview

This extension is useful for the following reasons:
  • URLs in email clients will typically open in Safari when opened. The Safari plugin enables your users to quickly receive access to apps that are tied to Okta.
  • URLs opened to Safari allows for deep linking which means the email receiver is sent directly to the page that the sender intended. The Safari plugin adds to the user experience by making it easy for users to access those deep links.
  • Users can now perform Service Provider-initiated logins to SWA and SAML apps in Mobile Safari.
