Devices

The Devices page displays all the devices in your orgAn abbreviation of organization, but can also be thought of as a company. A company that uses Okta as their SSO portal is generally referred to as an org. As an administrator, you decide how Okta should be displayed and/or integrated with your org. that have at one time been enrolled in Okta Mobility Management, with numerous options for display and depth of information about each device. The top-level Dashboard provides a wide overview of the device population, while the lower half of the page displays device-specific data and a reporting option with .csv file output.

Viewing Device data

By default, all devices are displayed. Use the left-side pane filter to display individual device information by: OS, Status, Device Activity, or Device Security.

Use the Search text box to search by user attributes (username, email, first name, or last name) or device attributes.

Reporting

For granular data about each device, you can run a report by clicking the Download CSV button. The resulting .csv file provides a very comprehensive output of each device, including device status, serial number, platform, device capacity (MB), and available device capacity (to name a few).

Deprovisioning and Factory Reset By User

Deprovisioning removes all managed apps and configuration and unenrolls the device from Okta Mobility Management.

Note: Deprovisioning does not require end-user confirmation. After you deprovision users, their managed apps and configuration are removed. They see the effects of the deprovisioning the next time they sign in.

To perform deprovisioning or factory resets by user:

  • For Android devices, the Admin resets the passcode.
  • For iOS devices the Admin clears the passcode and the end user needs to set up a new one themselves within one hour.

Note that this feature has a minimum requirement of Okta Mobile 1.2.8 for Android or Okta 4.2 for iOS.

To reset or clear a device passcode (depending upon device type) remotely:

  1. Select the Device Attributes page of the device whose password you want to clear or reset.
  2. Select Reset Passcode (for Android) or Clear Passcode (for iOS) (the button label changes according to device type).

For iOS devices, click Clear Passcode to confirm. The end user then has one hour to set a new passcode.

Jailbreak/Root Detection

Okta provides Jailbreak/Root detection for OMM devices.

About Jailbreak/Root Detection:

  • Jailbreak Detection: If an iOS device has been jailbroken, it means your end user has installed unsupported software on their device to either run custom software without Apple's permission, or to pirate copies of software or in appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. purchases.
  • Root Detection: If an Android device has been rooted, while it doesn't mean the end user has necessarily done anything inappropriate with their device, it does mean they have access to alter the firmware and/or OS that Google and the manufacturer/carrier shipped with the device.
