View Device Data

From the Devices > Overview page, you can view and search the status of the device population and deprovision or wipe (factory reset) them. You can also monitor all devices to see if they remain unencrypted, have been jailbroken or rooted. Clicking the buttons under the Actions column allows you to deprovision OMM functions or wipe the device directly from this overview page.

Wipe Company Data

The Wipe Company Data option removes all managed apps and configurations, and unenrolls the device from OMM. Devices wiped of company data are listed as Deprovisioned under the Status column. When an end user is deactivated from Okta, their OMM-enrolled devices are automatically deprovisioned, and all company-managed apps and data are removed from the device in OMM. The device appears in the device report as Deprovisioned. If the user is reactivated later, they must re-enroll their device.

To wipe a device of its company data, go to Devices > Overview > device and click Wipe Company Data in the Actions column.

End user deactivation

Rather than having end users remove OMM from their own devices, we recommend that you first wipe all company data from the device. If an end user unenrolls themself from OMM, the Okta servers may never know of the removal (due to a poor network or a device that is off-line). This creates a state where a device is enrolled but deprovisioned. This behavior can exist for both rooted and non-rooted devices.

Note: Wiping company data does not require end-user confirmation. After you wipe a device of company data, managed apps and configuration are removed from it. End users see the effects of the wipe the next time they sign in.

Wipe company data from the People Page

  1. Go to Directory > People.
  2. Select the user whose device you want to wipe of company data.
  3. Select the Devices tab.
  4. Search for the relevant device(s).
  5. In the Actions column, click Wipe Company Data  to perform a selective wipe of only Okta-managed native apps, retaining personal apps, content, and settings.

Deprovision or Wipe from the Device Page

1. Go to Devices > Overview.

2. Select or search for the user or device you want to selectively or fully wipe.

4. In the Actions column:

  • Click Wipe Company Data  to perform a selective wipe of only Okta-managed native apps, retaining personal apps, content, and settings.
  • Click Wipe All Device Data to perform a full wipe of all apps and data from the selected devices, restoring them to factory settings. You can disable this option through mobile policy rules (Devices > Mobile Policies). For details, see Disable Device Wipe permission.
Jailbreak/Root Detection

Okta provides Jailbreak (iOS) and Root (Android) detection for OMM devices.

  • Jailbreak Detection: If an iOS device has been jailbroken, it means your end user has installed unsupported software on their device to either run custom software without Apple's permission, or to pirate copies of software or in-appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. purchases.
  • Root Detection: If an Android device has been rooted, while it doesn't mean the end user has necessarily done anything inappropriate with their device, it does mean they have access to alter the firmware and/or OS that Google and the manufacturer/carrier shipped with the device.

Admins can quickly see if a device has been jailbroken or rooted directly from the Dashboard of the Devices page. Clear or reset device passcodes

When passwords are lost or need to be reset for any reason:

  • For Android devices, the adminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button on the upper right side of the My Applications page. resets the passcode.
  • For iOS devices, the admin clears the passcode and the end user must set up a new one themselves within one hour.

Note: This option has a minimum requirement of Okta Mobile 1.2.8 for Android or Okta Mobile 4.2 for iOS.

  1. Go to Devices > Overview.
  2. Click the device whose password you want to clear or reset.
  3. On the Device Attributes page, click Device Actions on the upper-right side of the page.
  4. Click Clear Passcode (for iOS) or Reset Passcode (for Android) .

iOS

For iOS devices, click Clear Passcode to confirm. From that point on, the end user has one hour to set a new passcode.

Android

For Android devices, enter the new passcode you want to assign to this device, then click Reset Passcode.

Remote Locking

You can remotely lock a device. This option is useful when devices appear to be in an uncertain state but wiping (factory reset) is not warranted.

  1. Go to Devices > Overview.
  2. Click the device you want to reset.
  3. On the Device Attributes page, click Device Actions on the upper-right side of the page.
  4. Click Remote Lock.
  5. (Optional) Send a message and contact number to the device. For example, "Please return this phone to Tom B" and a contact number.
  6. Click Remote Lock.
Reporting

For granular data about each device, you can run a report by clicking Download CSV . The resulting .csv file provides a very comprehensive output of each device, including device status, serial number, platform, device capacity (MB), and available device capacity (to name a few).
