Manage dashboard tabs for end users

Managing dashboard tabs allows you to create dashboard tabs for end usersIn Okta literature, we generally refer to "end users" as the people who have their own Okta home page (My Applications), using chiclets to authenticate into all of their apps. End users do not have any administrative control. When we refer to "users" we are generally referring to the individual(s) who have administrative control. and choose the apps that you want to appear on them.

Overview
  • Create up to four dashboard tabs.
  • Drag and drop apps across and within tabs.
  • Preview individual end users' dashboard to see their assigned orgThe Okta container that represents a real-world organization.-managed apps and the tabs on which they're located.
  • Choose whether to apply the configured dashboard to new, non-activated users or to all activated users.
Caveats
  • To configure this feature, admins must have super adminAn abbreviation of administrator. This is the individual(s) who have access to the Okta Administrator Dashboard. They control the provisioning and deprovisioning of end users, the assigning of apps, the resetting of passwords, and the overall end user experience. Only administrators have the Administration button on the upper right side of the My Applications page., org admin, or mobile admin permissions.
  • This feature is not available in the Okta Developer Console.
  • You cannot use this feature to assign or delete apps. For those operations, see The Applications Page.
Options
Apply a dashboard to new, non-activated end users

In their first sign in to Okta, all new end users that you activated after you perform this procedure will see the tabs you created and the appsEnd users see only the org-managed apps that have been assigned to them. you placed on them. This allows you to manage end users' first day experience. Once signed in, end users can move apps within and between tabs, create their own tabs and add personal apps, and delete tabs. TipYou can apply different dashboards to different sets of new, unactivated end users. For example, you can configure a dashboard for one set of new end users, and then activate those end users. Then you can configure a different dashboard for another set of new end users, and then activate them. Once activated and signed in to Okta, the two sets of new end users will see a different dashboard.

  1. Go to Settings > Customization.
  2. Click End-user dashboard.
  3. Configure the dashboard. You can do any of the following:
    • Create a tab. Click the + sign, enter a name, and then click Save. You can create up to 3 tabs.
    • Rename a tab. Hover over a tab and click the pencil icon.
    • Add apps to tabs. Drag and drop apps onto tabs or move apps within the current tab.
    • Delete a tab. First remove all the apps out of the tab, and then click Delete.
    • Preview a users' homepage. For details, see here.
  4. Click Apply dashboard to new users.
  5. Activate and deactivate usersend users.

Apply a dashboard to all activated end users

Click: Important to know before you begin
  • Applying a dashboard to all end users overrides the current dashboard configuration for all activated end users in your org, and all their appAn abbreviation of application. Essentially, it is a web-based site used to perform any number of specific tasks, and requires authentication from end users by signing in. chicletsThe "buttons" that appear on an end user's Home page and represent each application they wish to access through Okta. Clicking the chiclet allows the end user to instantly sign in and authenticate themselves into their chosen app. are moved to new locations. End users' personal apps are consolidated onto a new tab titled My Apps. These changes cannot be undone.
  • Because this option can significantly change the appearance and organization of end user dashboards, Okta recommends that you notify your end users in advance about the changes they are going to see.
  • It takes some time for your dashboard updates to complete. During the process, buttons are disabled and the dashboard cannot be edited.
  • To confirm that the new dashboard was applied, Okta automatically sends an email to the admin who applied the dashboard to all end users. If any apps are experiencing errors, a unique ID is provided in the email that you can give to Okta Support for remediation.
  • A banner appears on end users' Home page when their admin has customized their dashboard.

  1. Go to Settings > Customization.
  2. Click End-user dashboard.
  3. Configure the dashboard. You can do any of the following:
    • Create a tab. Click the + sign, enter a name, and then click Save. You can create up to 3 additional tabs.
    • Rename a tab. Hover over a tab and click the pencil icon.
    • Add apps to tabs. Drag and drop apps within the current tab or to another tab.
    • Delete a tab. First remove all the apps out of the tab, and then click Delete.
    • Preview a users' homepage. For details, see here.
  4. Click Apply dashboard to all users.

Preview an end user's dashboard

You can use the Preview field at the bottom of the page to search for end users and preview their dashboard. Having visibility into end users' dashboards can help you answer questions and resolve issues. You can use this option at any time, independently of the Apply dashboard . . . functions.

Note: Only org-managed apps are visible to admins; admins cannot see self-service apps or end users' personal tabs.
