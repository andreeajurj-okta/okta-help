Appearance

Appearance

Admins can specify the look and feel of their end users' My Applications or "home" page. Use the Appearance page to customize how much or how little of Okta is shown for an org.

Display Options

The elements under Display Options have a large impact on how your end users experience their My Applications home page, as well as the admin user experience.

In the Display Options section, click Edit and specify the following two display options:

  • Logo URL: If you choose to upload your org's logo (as explained in the following section), you can have the logo link to your company's website. Specify the URL you want to link to in this field.

    A logo must be uploaded before the link can become active. A logo file must be in .jpg, .png, or .gif format. The maximum file size is 100kB, and the maximum dimensions are 3000 x 500px.

    Click Upload Logo in the Organization Logo section below, and then click Choose File to find the logo file you previously created. Click the Upload Logo button. The image appears as a thumbnail which displays when your users sign in.

  • Enable Okta Home Footer: Specify whether you want to the Okta footer to appear on your end users' home pages (True/False).

Click Save when done.

The New Sign-in Experience

What is the New Sign-in Experience?

Okta's new sign-in experience introduces a new look and the following new functionality:

  • Adding Links to Helpful Resources
  • Managing Duo Devices
  • Unrecognized Org Security Message
  • Changes to MFA Challenge
  • Sign-In Page Optimization for Embedded Browser Flows

The new sign-in experience removes obstructive banners and optimizes text size to ensure that users no longer need to scroll when logging in on screens smaller than 700 px wide, such as embedded browser flows.

Note: The new sign-in experience is powered by the Okta Sign-In Widget, a full-featured, embeddable JavaScript toolkit for authenticating users, performing multi-factor enrollment and verification, recovering forgotten passwords, and unlocking accounts.

Requirements

  • For the new sign-in page to display correctly, your browser must be at least 750px in height.
  • To enjoy the new sign-in experience in Windows Internet Explorer, you must use version IE8 or later.

Procedures

Enable the New Sign-in Experience

  1. In New Sign-In Configuration, click Edit and then click Enable.
  2. In Sign-In Background Image, click Browse to locate the background image you want to appear in sign-pages, and then click Upload Image. Or, if you want to restore the default image, click Use Default.
  3. Click Save.

Adding Links to Helpful Resources

Click Need help signing in? to expand the sign-on widget and display links to additional help resources.

Managing Duo Devices

The new sign-in experience uses the Duo Web SDK instead of direct API integration. This change allows admins to leverage the latest updates from Duo, such as the ability to manage Duo devices directly from the login page. And if the Duo self-service portal is enabled for your org, your users can also add, update, and remove authentication methods by clicking the Manage Devices button. For more information, see Configuring Duo Security.

Changes to MFA Challenge

If multifactor authentication (MFA) is configured for your org, please note the following changes:

  • MFA options are now accessed from a drop down menu.
  • There is now a Trust this device option on the MFA challenge screen. Selecting the option configures Okta to remember your end users' devices for the period of time specified in your org's Sign-On policy.
Organization Logo

One way you can customize your end users' My Applications or "home" page is to add the company logo for your org.

  1. Prepare a logo file. The file must be a .jpg, .png, or .gif, no more than100kB, with maximum dimensions 3000px x 500px.
  2. Under Organization Logo, click Upload Logo, and then browse to find the logo file you previously created.
  3. Click Upload Logo. The image appears as a thumbnail which displays when your users sign in.
Application Theme

Under Application Theme, click a thumbnail. Clicking Change Theme applies the selected color scheme to your end user's My Applications home page.

The selected theme appears with these colors when your end users sign in.
