Administer DataStores

Overview

The purpose of guide is to describe Access Gateway DataStores and how they can be created, used and managed.

What’s covered in this guide

  1. Concepts
  2. Adding Data Stores
  3. Configuring an Sql Database Datastore
  4. Configuring an LDAP Datastore
  5. Test using Simulations
  6. Managing Data Stores

Concepts

DataStores:

  • Augment Access Gateway sessions using external data stores.
  • Support bi-directional synchronization between Access Gateway and external data stores.
  • Support databases, including: MySQL, MS SQL, Oracle and Postgres.
  • Support LDAP. including: Any LDAP V3 compliant Active DirectoryActive Directory (AD) is a directory service that Microsoft developed for the Windows domain networks. It is included in most Windows Server operating systems as a set of processes and services. Initially, Active Directory was only in charge of centralized domain management., Oracle Internet Directory/Unified Directory, OpenLDAP and others.
  • May be added, modified, deleted and otherwise managed within the Access Gateway UI.
Adding Data Stores

To add a data store:

  1. Navigate to your Access Gateway InstanceAn instance, or computer instance, is a virtual machine (VM) or individual physical computer, used to host a software appliance..
  2. Select the Settings tab.
  3. Select the Data Stores pane.
  4. Select (+)> Sql Database or (+) > LDAP Database.

Test using Simulations

Data stores can be testing using simulations.

  • To test a database data store:

    1. In the row associated with database the data store click the simulate button.
    2. For each where clause field enter a value which should match the associated entry in the idP.
    3. Click Test to run the sum the simulation.
      Click Close to end.
    4. Examine the result which should return data from the data store for the provided key.

  • To test an LDAP datastore:
    1. In the row associated with the LDAP data store click the simulate button.
    2. For each LDAP field in the Matching Filter enter a value
    3. Click Test to run the sum the simulation.
      Click Close to end.
    4. Examine the result which should return data from the data store for the provided filter fields.

Managing Data Stores

After creation, data stores can be:

  • Activated and deactivated - Use the active/inactive () toggle to activate or deactivate a data store.

    Caution

    Deactivating an in use data store may cause unexpected issues.

  • Edit - Use the edit () button to change any of the values associated with a data store.
  • Deleted - Use the delete () button to delete an existing data store.

Caution

Deleting an in-use data store may cause unexpected result.  Before deleting a data store always remove all associated attributes before deleting the data store.
