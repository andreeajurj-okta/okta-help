Suspicious Activity Reporting

FF: REPORT_SUSPICIOUS_ACTIVITY_VIA_EMAIL


Suspicious Activity Reporting allows an end user to report unrecognized activity to their org admin from an account activity email notification.

Overview

When a user reports suspicious activity, admins can enable specific actions and audit system logs events to obtain further details about the activity reported.

There are two ways to access this feature from the admin console:

  • Navigate to Security > General, under Security Notification Emails.
  • View and select event status directly from the dashboard.

 

End-user experience

When this feature is enabled and email templates are configured to allow reporting, end users will have an option to report suspicious or unrecognized activity to their org admin from an email notification.

  • The link is only valid for 7 days after the email is sent and the action.
  • The link expires after the user confirms suspicious activity.

Before you begin

Note the following before you configure and enable this feature:

  1. You must be a super admin or org admin to configure this feature. All other admins have read-access only.
  2. In order for end users to report suspicious activity, ensure that at least one of the following email notifications are enabled:
    • New sign on notification email
    • MFA enrolled notification email
    • MFA reset notification email

If your org has customized email templates, they must be customized to include a button link for end-user reporting. Templates that are not customized include the Report button by default. To add this button, see step 6 in Configure Suspicious Activity Reporting.

 

Procedures

Configure Suspicious Activity Reporting

  1. From the admin dashboard, navigate to Security > General. The General Security page is displayed.
  2. Under Security Notification Emails, click Edit to configure this feature. The following settings are available:
    • New sign on notification email
    • MFA enrolled notification email
    • MFA reset notification email
    • Report suspicious activity via email

 

System Log events

If ThreatInsight actions are enabled, requests from malicious IP addresses will appear in the admin System Log, which can be accessed from the admin console menu or directly from the link provided in Okta ThreatInsight Settings.

Enter the following query to find these type of events in the system log: eventType eq "security.threat.detected"

 

Related links

When ThreatInsight actions are enabled, end users may sign in to their org as usual. If a sign-in attempt from a malicious IP address is detected and authentication requests are set to be blocked, the user receives an HTTP 403 error.
