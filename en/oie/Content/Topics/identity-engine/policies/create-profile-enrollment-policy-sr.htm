Create a Profile Enrollment policy for self-registration

This Profile Enrollment policy allows end users to self-register through a Sign-up link in the Okta Sign-in widget. By the time the user has authenticated into the app, their profile is created and they’ve been provisioned to the appropriate groups.

Org sign-up is controlled by the default Profile Enrollment policy. To enable sign-up for a specific app, but not the org, do not allow sign-up in the default policy. Create a new Profile Enrollment policy that applies to the app and allow sign-up there. These settings are described further in step 7.

  1. In the Admin Console, go to Security > Profile Enrollment.
  2. Select the default Profile Enroll policy, or, to create a new one, click Add New Profile Enroll Policy.
  3. Enter a new policy Name.
  4. On the policy page, click Manage Apps.
  5. On the Apps Using (Policy Name) page, click Add an App to this Policy.
  6. On the policy page, click More Options > Edit for the default rule.
  7. Indicate your conditions.
    • Prompt the user for data: Indicate which attributes appear in the Profile Enrollment form. When you add a field, its display name appears in Form Label. You can make fields on the enrollment form optional, but the end user will be required to enter a value if they are required by the user schema. If you want these new fields to persist, add them to Profile Editor as described in step 8.
    • Email Verification: Indicate whether email verification is required before access is granted.
    • Add the user to group: Indicate which group the user is automatically added to. To review groups in your org, click Go to Groups.
    • Sign-Up: This is where you can customize self-registration.
      • If you want users to self-register for your org, select this option in your default policy.
      • If you want users to self-register for the app but not the org, deny sign-up in your default policy. Then, create a new Profile Enrollment policy specifically for the app and select Sign-Up there.
  8. If you added fields to the enrollment form in the previous step, go to Directory > Profile Editor.
    • Find the new attribute, and then click its information icon.
    • In the User permission drop-down box, select Read - Write. End users who do not have write access to any required fields can’t complete self-registration.
    • Click Save Attribute. End users can modify this field after registration in their settings page.
  9. In the THEN Use the following inline hook condition, indicate whether you want to use an inline hook for registration. The hook will not run until the user has completed the enrollment form.
  10. Click Save.

End user experience

When Sign-Up is enabled, new users can click the Sign-up link in the Okta Sign-in widget. This launches the Create Account registration form, where they fill out the fields you configured in the Profile Enroll Policy. Okta automatically uses the email address as the end user’s username and primary email address.

  • Make email verification mandatory. After registering their information, users are immediately sent a link to verify their email address and complete the registration process. The activation email satisfies possession assurance through the email authenticator, and it carries over any factor verifications associated with the initial sign up. Additional authentication prompts may appear, depending on the reauthenticate settings in your sign-on policies. After satisfying all authentication requirements, end users are redirected to your app or to your org’s Okta home page.

  • Make email verification optional. After registration, end users are immediately redirected to your custom app/portal or to your org’s Okta home page. Okta then sends the user an email requesting verification of their email address.

Update a Profile Enrollment policy

  1. In the Admin Console, go to Security > Profile Enrollment.

  2. On the Profile Enrollment page, locate the policy, and then click the edit or delete icon in the Actions column. If you want to delete the policy but the icon is not available, this policy is assigned to an app. See step 3 for options.

  3. On the Policy page, you can:

    • Assign the policy to apps (or remove it) by clicking Manage Apps.

    • Change an app’s policy by clicking the tile menu and then selecting Apply a Different Policy.

    • View app settings by clicking the tile menu and selecting Go to App.

    • Change conditions in the rule by clicking the More Options menu and then selecting Edit. If you add required attributes to the Enrollment form, users will be prompted for this information during their next sign in attempt.
