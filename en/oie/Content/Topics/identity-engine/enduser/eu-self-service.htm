Self Service Account Settings

Account settings enables you to configure and manage profile information and security settings information.

Configuring My Profile settings

You can manage your profile information and specify the display language using the settings on the My Profile tab.

Change end user information

You can configure and change your own personal information that was collected during Okta enrollment.

  1. In the Okta End User Dashboard go to Account.
  2. Click My Profile.
  3. Click Edit to configure personal information.
  4. Continue by configuring the information in each field of the Profile:
    • Username (must be in the format of an email address)
    • First name
    • Last name
    • Primary email
    • Secondary email
    • Mobile phone
    • Favorite Number (for example, Office floor number)
  5. Click Save.

Change display language preference

Users can select their language preference for interface. This preference is used in end user pages as well as in emails and notifications.

  1. In the Okta End User Dashboard go to Account Settings.
  2. Click My Profile.
  3. In Display Language, from the Display Language drop-down, select your language preference.
  4. Click Save.

Security

Security settings enable you to manage your password settings and their identification credentials used for logging into their account and for verifying their identity.

Self service password settings

You can perform a self-service password reset by first verifying your identity using one of the following authenticators: security question, email, voice call, SMS, webauthn and Okta Verify. Once your identity is verified, you can change your password.

  1. In the Okta End User Dashboard navigate to Settings > Security.
  2. In the list of enrolled credentials, click Update.
  3. Verify your identity using an authenticator that was previously specified in the assurance policy..
  4. In the Change Password window enter the following:
  • Enter current password
  • Enter new password
  • Confirm new password

5. Click Change Password.

 

 

Security Question

Customize the security question that is used to enable sign in.

Note

The admin must enable the Security Question factor on the Admin Console.

When configuring your security question use the following guidelines:

  • The answer to a security question must be at least four characters long.
    • a higher minimum length requirement can be set by adjusting the appropriate password policy's Answer Complexity value (Security > Authenticator > Password in the Okta Admin Console. See Password Policies).
  • The answer to a security question cannot be your password or user name.
  • The answer to the security question cannot contain part of the question.

To configure your security question:

  1. In the Okta End User Dashboard go to Settings > Security.
  2. Click the Security tab.
  3. In the list of Authenticators, click Set up beside the security question.
  4. Enter an answer, and then click Save.

Removing an Authenticator

  1. In the Okta End User Dashboard go to Settings > Security.
  2. In the list of enrolled credentials, click Remove beside the authenticator that you want to remove.
  3. Verify your identity using an authenticator that was specified in the assurance policy.
  4. The enrolled credential is removed from the Settings page.

 

Forgotten Authenticator Settings

Multifactor Authentication (MFA) is an added layer of security used to verify your identity when you sign in to an application. Users are prompted to confirm their credentials with factors both when signing in to Okta and when accessing an application.

Note

End users are not automatically signed out of an application when their end user's session expires until they perform an action on end user dashboard.

An Okta admin can configure MFA at the organization or application level. If both levels are enabled, end users are prompted to confirm their credentials with factors both when signing in to Okta and when accessing an application.

When enabled by the admin, you can set up additional authenticators such as:

  • Security Key or Biometric
  • Okta Password
  • Phone Number
  • Primary Email Address
  • Security Question

 

Security Key or Biometric

The Security Key or Biometric authenticator follows the FIDO2 Web Authentication (WebAuthn) standard.

Once this authenticator is enabled and set to required or optional, end users mayselect it when signing in and set it up so it may be used for additional authentication.

The Security Key or Biometric authenticator supports two authentication methods:

  1. Security keys such as YubiKeys or Google Titan.

  2. Platform authentication that is integrated into a device and uses biometric data or a PIN such as Windows Hello or Apple Touch ID.

 

Okta Password

The Password authenticator consists of a string of characters that can be specified by users or set by an admin. Admins can also change the password complexity requirements and recovery settings by editing this authenticator.

Once added, the Password Authenticator may be configured further as part of the Password Policy. Password policies and rules are now located under Security > Authenticators > Password.

To configure a Password Policy:

  1. In the list of added Authenticators, find the Password entry and click Actions > Edit.
  2. Add a new policy or edit an existing policy.

  3. Set the following options for user passwords as needed:

    Authentication providers

    • Apply the password policy settings to Okta users.

    Password settings

    • Minimum length: Require a minimum number of characters.

    • Complexity requirements: Enforce various character requirements or exclude a user’s name.

    • Password age: Reuse of passwords for a designated number of password instances and password expiry time.

    • Lockout: Set the number of times an incorrect password may be entered before the account is locked.

    • Account is automatically unlocked after __ minutes: The account is automatically reset without additional action from users after the number of minutes that you specify. The minimum setting is one minute. The user password remains unchanged when the account is unlocked.

    Self-service password recovery

    • Admins can specify authentication requirements for the user self-service password recovery. Choose from SMS, Email, Security Question, or Voice Call for the user to reset their password.

    • If Email is selected in the password policy configuration, a password reset email is sent to the user’s primary email address. This requires the Email authenticator and uses both an email magic link and OTP code.

Phone number

SMS and Voice Call are now merged into the Phone authenticator, which can be accessed from Security > Authenticators > Phone.

  • Users can enroll in the Phone authenticator using either SMS or Voice Call methods.

  • Users can authenticate using either method.

  • Phone numbers with extensions will not have Voice methods available.

  • Users can use either SMS or Voice Call to enroll in the Phone authenticator.

  • Once a user enrolls in SMS for example, the user is also enrolled in Voice. This also applies if the user enrolls in Voice Call, they are also enrolled for SMS. This allows the user to use Self-Service Password Recovery for both SMS and Voice Call when enabled.

  • SMS methods are not available for phone numbers that have phone extensions.

After configuring this factor, users signing in to Okta see that extra verification is required.

If Phone Authentication is selected they will be instructed to input a mobile phone number where a security token will be sent via SMS or a call will be made to the number with a voice provided 6-digit code. Once the user enters the valid code then access is allowed.

 

Primary Email Address

The Email authenticator allows users to authenticate successfully with a token (referred to as an Email Magic Link) that is sent to their primary email address. Once a user receives the token, they either follow the link that contains the embedded token or alternatively use a 6-digit code in the email to authenticate.

Email magic link and email OTP are both supported. Email magic links and 6 digit codes are valid for 5 minutes. This duration cannot be configured.

  1. In the Admin Console, go to Security >Authenticators.
  2. On the Authenticators tab, click Add Authenticator.

  3. Click Add for the Email authenticator.

 

Security Question

The Security Question authenticator can be managed and reset by admins. It will show up as required when end users enroll in MFA, even if it isn’t present in the MFA enrollment policy. You can disable it by deselecting Security Question under Authenticators > Password > Edit > Additional Self-service recovery.

Remember that a security question can’t be used in conjunction with a password to satisfy two-factor authentication in an app sign-on policy. Two-factor authentication in app sign-on policies requires two different factor types to be verified. Okta recommends against using security questions in an authentication flow.
