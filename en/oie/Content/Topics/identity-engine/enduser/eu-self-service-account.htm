Configure Settings

Manage your personal information, specify the display language, and change password settings on the Account page.

Enable Settings

Prior to configuring settings information, you need to verify your identity.

  1. In the End User Dashboard, go to Settings.
  2. Click Edit Profile.
  3. Enter your password and then click Verify.
  4. If multifactor authentication is configured, you are prompted to confirm your identity by using your second authentication factor.

Configure personal information

You can configure your personal information without having to contact an administrator.

Before you begin

Enable editing for the Account page. See Enable Settings

To configure personal information:

  1. In the End User Dashboard, go to Settings.
  2. In Personal Information, click Edit.
  3. Configure the information in each field:
    • Username (must be in the format of an email address)
    • First name
    • Last name
    • Primary email
    • Secondary email
    • Mobile phone
  4. Click Save.

Reset your Okta password

In the Account page, change your password without having to contact an administrator.

Before you begin

Enable editing for the Account page. See Enable Settings

To reset your Okta password:

  1. In the End User Dashboard go to Settings
  2. In Change Password enter your current and new password.
  3. Click the Change Password button to confirm the change.

Select your display language

Customize your Okta experience by selecting your language from a list of supported languages. After you sign in to Okta, you see the Okta dashboard in the language you chose. All emails generated by Okta will also be in your selected language.

Before you begin

Enable editing for the Account page. See Enable Settings

To select your display language:

  1. In the End User Dashboard go to Settings.
  2. In Display Language, click Edit. .
  3. Select a language from the drop-down list.
  4. Click Save.

Configure extra verification

Extra Verification settings enable you to set up additional authentication factors such as: security key or biometric, password authenticator, phone number or security question that is used for logging into your account and for verifying your identity.

Extra verification types

Extra verification types appear in Settings only if your administrator has configured additional authentication requirements (multifactor authentication) for your organization. For details, contact your administrator.

Extra Verification Description
Security key or biometric authentication

The Security Key or Biometric authenticator supports two authentication methods:

  • Security keys such as YubiKeys or Google Titan.
  • Platform authentication that is integrated into a device and uses biometric data or a PIN such as Windows Hello or Apple Touch ID.

Security Key or Biometric authenticator is also used with User Verification settings. User Verification is a FIDO2 setting in which an authenticator verifies if a user is authorized to use it. This verification can include a biometric challenge, PIN, or password in addition to tapping the device.
Password Authenticator

  • Users can select from SMS, Email, Security Question, or Voice Call for the user to reset their password.

  • If Email is selected in the password policy configuration, a password reset email is sent to the user’s primary email address. This requires the Email authenticator and uses both an email magic link and OTP code.
Phone number

SMS and Voice Call are now merged into the Phone authenticator.

  • Users can enroll in the Phone authenticator using either SMS or Voice Call methods.

  • Users can authenticate using either method.

  • Phone numbers with extensions will not have Voice methods available.

  • Users can use either SMS or Voice Call to enroll in the Phone authenticator.

  • Once a user enrolls in SMS for example, the user is also enrolled in Voice. This also applies if the user enrolls in Voice Call, they are also enrolled for SMS.

  • SMS methods are not available for phone numbers that have phone extensions.

To use Phone Authentication, input a mobile phone number where a security token will be sent via SMS or a call will be made to the number with a voice provided 6-digit code. Once the user enters the valid code then access is allowed.
Security question

All the following guidelines are required for security questions:

  • The answer to a security question must be at least four characters long; however, a longer length can be specified for recovery flows in a Group Password Policy.
  • The answer to a security question cannot be the user's password or user name.
  • The answer to the security question cannot be included in the question.

A security question can’t be used in conjunction with a password to satisfy two-factor authentication in an app sign-on policy. Two-factor authentication in app sign-on policies requires two different factor types to be verified. Okta recommends against using security questions in an authentication flow.

Start this task

 

To configure Security Question:

  1. In the End User Dashboard go to Settings.
  2. In Extra Verification, click Set up beside Security Question authenticator.
  3. Choose a security question, enter an answer, and then click Save.

 

To configure Phone Authenticator (SMS):

  1. In the End User Dashboard go to Settings.
  2. In Extra Verification, click Set up beside SMS authenticator.
  3. In the Set up page, click Setup.
  4. In the SMS page, from the drop-down list, select a country.
  5. In Phone number, enter the mobile phone number where you want your security tokens sent.
  6. Click Send Code button.
  7. Enter the 6 digit code that was sent to your phone.
  8. Click the Verify button.

To configure Phone Authenticator (Voice Call):

  1. In the End User Dashboard go to Settings.
  2. In Extra Verification, click Set up beside Voice Call authenticator.
  3. In the Set up page, click Setup.
  4. Enter the number for the mobile or landline phone on which you want to receive the call. An extension number can be entered for landline business phones, as illustrated in the sample image under Sign-In Experience.
  5. In Phone number, enter the mobile phone number where you want your security tokens sent.
  6. Click the Call button.
  7. The number is verified with a call and a Call is in progress... message appears. Enter the provided code into the Enter code box.
  8. Click Verify button.

 

To configure Security Key or Biometrics

  1. In the End User Dashboard go to Settings.
  2. In Extra Verification, click Set up beside Security Key authenticator.
  3. When prompted, enter your password and click Verify.
  4. From the drop-down list, select the authenticator option.
  • YubiKey
  • Apple Touch ID
