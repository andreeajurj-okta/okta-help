Build an _authping Flow

This is a Beta feature. To see about participating in this Beta program, please refer to the Beta Programs page.

_authping is a system Flow that allows the platform to “ping” an API and check for valid authorization. This acts as a sort of health check that runs when the connector is first authenticated, new connectors are added, or Flows are opened. The Flow must be named _authping in order to be recognized by the platform. Authping Flows are not displayed to the end user and are only used by the platform. It has no options or input fields.

  1. Create a new Flow under the Flows tab by clicking on + New Flow in the upper right corner of the page.

  2. At the far left of the page, click Authping.

  3. Click the Save button in the toolbar and name the Flow _authping.

  4. In the Flow description field type Connection health check, check the Save all data that passes through the Flow box, and click Save.

  5. Add a Flow Control Call Flow card as the first card in the Flow.

  6. Click on the Choose Flow button and select the http_helper Flow created previously. The card will automatically populate with the inputs defined in the http_helper Flow.

  7. Manually enter the relative url of the endpoint to use as a “ping”. The endpoint should require authorization from the server, and the ping response should contain a small packet of data including a standard HTTP status code.

  8. Manually enter a request method (GET, PATCH, POST, PUT, or DELETE).

  9. Enter any additional information required for the call in the query, headers, or body field. You may need to add an Object Construct card before the Call Flow card to build out any necessary objects.

  10. For the outputs on the Child Flow card, you’ll need to define the same keys as on the http_helper Flow’s outputs:

    • status_code (Number)

    • headers (Object)

    • body (Object)

      On the body object, you’ll need to further define a message key as defined in the http_helper Flow outputs.

  11. Add an Object Construct card after the Call Flow card.

  12. Click on the area that reads Click or drop here to create and add the label working.

  13. Change the data type to True/False, and select True from the resulting dropdown.

  14. Add another input field on the Construct card using dot notation labeled serviceResponse.statusCode.

  15. Drag the status_code output field from the Call Flow card into the serviceResponse.statusCode field.

  16. Add another input field on the Construct card using dot notation labeled serviceResponse.body.

  17. Manually add a success message to the serviceResponse.body field. For example, This account is successfully connected.

  18. Rename the output on the Construct card to valid_connection.

  19. Add another Object Construct card.

  20. Click on the area that reads Click or drop here to create and add the label working.

  21. Change the data type to True/False, and select False from the resulting dropdown.

  22. Add another input field on the second Construct card using dot notation labeled serviceResponse.statusCode.

  23. Drag the status_code output field from the Call Flow card into the serviceResponse.statusCode field.

  24. Add another input field on the second Construct card using dot notation labeled serviceResponse.body.

  25. Drag the message key from the body output on the Call Flow card into the serviceResponse.body input on the second Construct card.

  26. Rename the output on the second Construct card to invalid_connection.

  27. Add a Branching Assign If card.

  28. Drag the status_code output field from the Call Flow card into the value a field.

  29. Change the comparison dropdown to equal to.

  30. In the value b input, manually enter the expected status code of a valid request. This is usually 200 or 201.

  31. Drag the valid_connection output from the first Construct card into the value if true input on the Assign If card.

  32. Drag the invalid_connection output from the second Construct card into the value if false input on the Assign If card.

  33. Rename the output on the Assign If card to authping_return.

  34. Click the Add Outputs button on the Return Outputs card.

  35. Click the Add Field button to add a single output field named output and type object and click Save.

  36. Drag the authping_return field from the Assign If card to the output field on the Return Outputs card.

  37. Save and Test your Flow.
